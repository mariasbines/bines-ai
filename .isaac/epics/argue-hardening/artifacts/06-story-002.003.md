# Story 002.003: Refusal template + system-prompt tightening + voice rubric

## Description

The voice layer of the filter. Three small deliverables in one focused story:

1. The locked refusal text + the single-event SSE stream builder that renders it in the chat response shape.
2. A tightening of the existing Sonnet system prompt to declare the Q6 off-brand categories as out-of-scope with in-voice deflection language — belts the classifier (if Haiku misses, Sonnet has its own guidance).
3. A voice-check rubric sibling to the existing `docs/chat-voice-check.md` that Maria can use to verify the refusal text and the tightened prompt behave correctly.

This story does not call Haiku and does not touch the chat route. It ships the refusal *module* that 002.004 will wire up. Pulling it out of 002.004 keeps the voice-layer concerns isolated from the classifier-plumbing concerns — easier to tune refusal copy without re-running classifier tests.

## Acceptance Criteria

- [ ] AC-001: `src/lib/argue-filter/refusal.ts` exports:
  - `REFUSAL_TEXT: string` — the locked text: `"not my lane — maria doesn't have a public position on this, and i don't invent them. what else have you got?"`
  - `refusalEventStream(): ReadableStream<Uint8Array>` — returns a single-event SSE body matching the Anthropic SDK stream shape (content_block_delta → message_stop), UTF-8 encoded
- [ ] AC-002: `refusalEventStream()` output, when consumed by the same client parsing code used for normal chat streams, produces the refusal text as the assistant's reply without errors. (Unit test asserts the stream shape byte-by-byte.)
- [ ] AC-003: `src/lib/chat/system-prompt.ts` is modified to append an "out-of-scope topics" section that:
  - Names the seven Q6 off-brand categories verbatim (electoral politics; hot-button social issues; race-as-identity-politics; religion; named real people outside Maria's published circle; family of Maria beyond what's on site; conspiracy / crypto hype)
  - Instructs the model to deflect with language consistent with `REFUSAL_TEXT` when any such topic arises, without lecturing
  - Does NOT duplicate `REFUSAL_TEXT` verbatim (prompt instructs the *shape* of the refusal, doesn't hard-code the exact string — that's the classifier/route's job)
- [ ] AC-004: The existing `src/lib/chat/__tests__/system-prompt.test.ts` is updated to assert the new section is present (keyword-based assertions: each of the seven category names, plus the deflection-shape instruction).
- [ ] AC-005: `docs/argue-voice-check.md` exists as a sibling to `docs/chat-voice-check.md`. It contains:
  - A 10-prompt probe set covering each of the seven off-brand categories (at least one prompt per category)
  - The expected behaviour for each prompt (deflection in-voice, not substantive engagement)
  - A rubric for judging the refusal: (a) lowercase opener, (b) no corporate hedging, (c) no apology-stacking, (d) carries a question-hook ending where natural, (e) doesn't lecture
  - Instructions for running the probe against a preview deploy post-002.004
- [ ] AC-006: `REFUSAL_TEXT` is British English, lowercase, and passes the rubric criteria in AC-005. (This is a meta-assertion; the rubric exists for future tuning.)
- [ ] AC-007: `src/lib/argue-filter/__tests__/refusal.test.ts` exists and covers: text constant value, stream shape (at least one content_block_delta event containing the refusal text, followed by a message_stop), stream is UTF-8 encoded, stream is readable to completion.

## Technical Tasks

1. Create `src/lib/argue-filter/refusal.ts` with `REFUSAL_TEXT` const and `refusalEventStream()` function. The stream builder should construct SSE events matching Anthropic's streaming format (event name + JSON data per event, separated by `\n\n`). Use `TextEncoder` for byte conversion.
2. Create `src/lib/argue-filter/__tests__/refusal.test.ts` with the assertions from AC-007.
3. Modify `src/lib/chat/system-prompt.ts` — append the out-of-scope section as described in AC-003. Keep prose in Maria's register.
4. Modify `src/lib/chat/__tests__/system-prompt.test.ts` — add assertions for new section presence.
5. Create `docs/argue-voice-check.md` — 10-prompt probe, expected-behaviour table, rubric, run-against-preview instructions.

## Scope rules

- Do NOT call Haiku or any classifier in this story. This is voice-layer only.
- Do NOT modify `src/app/api/chat/route.ts`. Route integration happens in 002.004.
- Do NOT vary the refusal text by category or context in v1. One string. Rotation / variation is a post-launch follow-up.
- Do NOT include `REFUSAL_TEXT` verbatim inside the system prompt — Sonnet's prompt describes the *shape*, not the exact string. Single source of truth lives in `refusal.ts`.
- Do NOT add new antipatterns or voice rules beyond the off-brand list — the existing system prompt's rules stand.
