# Story 002.005: Launch QA + privacy disclosure update + soft share

## Description

The finish line. Four deliverables:

1. **Privacy-disclosure update** — the existing chat-interface disclosure in `src/components/ChatInterface.tsx:126` says *"this conversation is not stored. what you type is sent to Anthropic."* That becomes false the moment argue-log ships. This story updates the text in the same PR as any merge that carries argue-log to production (mitigates ARGUE-PRV-002). Note: dev-branch preview deploys already stream logs once 002.001-004 merge there; the hard rule is that **no production release of logging ships without the updated disclosure**. Since master is frozen until launch QA, the same-PR constraint binds at the dev→master merge point.
2. **Voice-check sweep** — run the 10-prompt probe set from `docs/argue-voice-check.md` against the integrated preview deploy, capture results, tune prompt/refusal if needed.
3. **Ops doc walkthrough** — Maria reads `docs/argue-log-ops.md` end to end; any ambiguity → fix the doc, not the memory.
4. **Soft share + monitoring** — once previous items are green, `/argue` can be shared with a small circle for a week. Admin view reviewed at end of week; if no voice regressions, `/argue` is safe to share broadly.

This is a verification-and-cutover story, not a code-writing story. Minimal mechanical change (the privacy text tweak); the bulk is manual QA + ops review. A week of post-merge soft-share monitoring lives inside this story's lifetime rather than as a new story — matches the elicitation Q9 "ship order: log first, filter second" intent.
