# Story 002.001: Conversation log + cleanup route

## Description

The foundation story. Builds the private Blob storage for `/argue` conversation logs, the salted-IP hasher, the zod schema, the daily-sweep cleanup route, the cron schedule, and the operations runbook. Nothing else in the epic can land without this. Risk-mitigation density is the highest in the epic (8 risks mapped here — see §Risk mitigations).

No route integration in this story — that's 002.004. This story ships the *library* that the route will call. It ships production-ready with its own tests and documentation.

## Acceptance Criteria

- [ ] AC-001: `src/lib/argue-log/schema.ts` exports `ARGUE_LOG_ENTRY` zod schema and `ArgueLogEntry` type. Schema matches the shape in `03-architecture.md §Components` exactly, including `schema_version: 1`, `verdict`, `refused`, `latency_ms`, and salted `ip_hash` fields.
- [ ] AC-002: `src/lib/argue-log/hash.ts` exports `hashIp(ip, salt) → Promise<string>`. Uses Web Crypto `crypto.subtle.digest('SHA-256', salt+ip)`. Returns lowercase 64-char hex. Edge-runtime compatible (no Node-only APIs).
- [ ] AC-003: `src/lib/argue-log/storage.ts` exports: `appendArgueLog(entry, options?)`, `listArgueLogDays()`, `readArgueLogDay(day)`, `deleteArgueLogDay(day)`. Validates entry against `ARGUE_LOG_ENTRY` before writing; throws on malformed input. Throws clean error if `BLOB_READ_WRITE_TOKEN` unset. Uses `import 'server-only'`.
- [ ] AC-004: Storage writes to Blob prefix `argue-log/YYYY-MM-DD.jsonl` using the get-then-concat-then-put pattern from `src/lib/push-back/storage.ts`. Same race tradeoff, accepted for v1.
- [ ] AC-005: `deleteArgueLogDay(day)` validates `day` against strict regex `^\d{4}-\d{2}-\d{2}$` before constructing the blob key; rejects non-matching input. Constructs key internally — no user input is concatenated into the blob path (mitigates ARGUE-SEC-008).
- [ ] AC-006: `src/app/api/argue-log/cleanup/route.ts` — edge runtime, `GET` handler. Requires `Authorization: Bearer <CRON_SECRET>`. Missing or wrong secret returns 401. Timing-safe comparison on the secret.
- [ ] AC-007: Cleanup route iterates `argue-log/` prefix via `listArgueLogDays()`, filters to days strictly older than 90 days (computed from current UTC date), calls `deleteArgueLogDay()` for each. Returns JSON `{ deleted: string[], skipped: number }`.
- [ ] AC-008: `vercel.json` contains `{ "crons": [{ "path": "/api/argue-log/cleanup", "schedule": "30 3 * * *" }] }`. If `vercel.json` does not exist, create it with only this field.
- [ ] AC-009: `docs/argue-log-ops.md` exists and documents: (a) salt-rotation quarterly procedure using `ARGUE_LOG_IP_SALT_CURRENT` + `ARGUE_LOG_IP_SALT_PREVIOUS` (two-env-var pattern), (b) cron-secret rotation annually, (c) manual-sweep curl command for forcing a cleanup run, (d) Vercel env-var setup for all four new vars (`FILTER_MODEL`, `ARGUE_LOG_IP_SALT_CURRENT`, `ARGUE_LOG_IP_SALT_PREVIOUS`, `CRON_SECRET`), (e) warning that Production `Deployment Protection` must never be toggled off (ARGUE-SEC-005 mitigation).
- [ ] AC-010: Grep audit — no test and no source file emits a Blob URL to console or response body. Verified by unit test that captures `console.log`/`console.error` calls during append/list/read/delete and asserts no `https://*.public.blob.vercel-storage.com` substring in any captured output (mitigates ARGUE-SEC-001).

## Technical Tasks

1. Create `src/lib/argue-log/schema.ts` — zod schema + inferred types per architecture doc.
2. Create `src/lib/argue-log/hash.ts` — Web Crypto SHA-256 hash of `salt + ip`. Pure function, no env reads.
3. Create `src/lib/argue-log/storage.ts` — modelled on `src/lib/push-back/storage.ts`; implement `appendArgueLog`, `listArgueLogDays`, `readArgueLogDay`, `deleteArgueLogDay`. Mirror the get-then-concat-then-put pattern. Schema-validate entry in `appendArgueLog` before writing.
4. Create `src/app/api/argue-log/cleanup/route.ts` — edge runtime GET handler with `Authorization: Bearer` check, age filter, batch delete via `deleteArgueLogDay`.
5. Create `vercel.json` (or modify if present) with cron schedule entry.
6. Create `docs/argue-log-ops.md` — runbook covering the five topics in AC-009.
7. Create unit tests:
   - `src/lib/argue-log/__tests__/schema.test.ts` — round-trip parse; rejects malformed.
   - `src/lib/argue-log/__tests__/hash.test.ts` — determinism (same inputs → same output), salt variance (different salts → different outputs), hex-length = 64, lowercase.
   - `src/lib/argue-log/__tests__/storage.test.ts` — mock `@vercel/blob` (pattern from push-back tests), cover: append on empty day, append on existing day (concat), list days sorted desc, read day parses JSONL lines, delete day, missing token, malformed-entry rejection, URL-leak guard.
   - `src/app/api/argue-log/cleanup/__tests__/route.test.ts` — missing auth → 401, wrong secret → 401, correct secret → deletes eligible days only, returns summary. Path-traversal safety (non-regex-matching keys skipped).

## Risk mitigations (from 04-assessment.md)

- **ARGUE-SEC-001** Blob URL leakage → AC-010 (grep audit in tests)
- **ARGUE-SEC-003** Salt misconfiguration → AC-009 (ops runbook), plus route-level fail-loud in 002.004
- **ARGUE-SEC-004** Cron secret leak → AC-006 (timing-safe comparison + bearer-only)
- **ARGUE-SEC-006** Stored content exfiltration → AC-003 (token boundary), AC-010 (URL never emitted)
- **ARGUE-SEC-008** Path traversal on cleanup → AC-005 (regex validation before key construction)
- **ARGUE-PRV-001** GDPR on logs → AC-004 (90-day retention via cron — see 002.005 for privacy-notice update)
- **ARGUE-OPS-001** Missed cron → AC-008 (daily schedule) + manual-sweep command in AC-009
- **ARGUE-OPS-003** Write race → accepted v1 per scope rule
