**L-001 · Vercel cron runs only on Production — document in ops**

- Severity: LOW
- File: Plan Step 7 (`docs/argue-log-ops.md`)
- Details: Vercel cron schedules fire only on the Production deployment, never on previews. The plan's cron schedule in `vercel.json` will not run on `dev`-branch previews; Maria's first confirmation that cleanup works happens post-dev-to-master merge. Plan text should surface this explicitly in the ops doc's env-var setup section.
- Suggested fix: Add a one-sentence note in ops doc §Env-var setup: *"Cron only fires on Production. Preview deploys don't auto-sweep — use the manual curl command from §3 to test."*
- Impact: None on code; 5-line doc addition during implementation.

**L-002 · `list()` prefix-match precision for `readArgueLogDay` and `deleteArgueLogDay`**

- Severity: LOW
- File: Plan Step 4
- Details: `list({ prefix: 'argue-log/2026-04-24.jsonl' })` could theoretically match `argue-log/2026-04-24.jsonl.backup` or similar if anyone ever creates sibling files. Currently no mechanism creates such siblings, but a defensive equality check (`b.pathname === filename`) inside the loop would harden against drift.
- Suggested fix: In `readArgueLogDay` and `deleteArgueLogDay`, filter `existing.blobs` with `b.pathname === filename` before fetching/deleting. Two extra lines.
- Impact: Defensive. Not required for correctness today; cheap insurance.

**L-003 · Blob-append race — same accepted v1 tradeoff as push-back**

- Severity: LOW
- File: Plan Step 4
- Details: Same get-then-concat-then-put race documented in push-back. At expected chat volume it's irrelevant. Plan acknowledges this and documents the migration path (per-turn files) if traffic warrants. No change needed.
- Suggested fix: None.

**L-004 · Timing-safe compare limited to BMP-range chars**

- Severity: LOW
- File: Plan Step 5
- Details: `charCodeAt` returns the UTF-16 code unit, which is fine for ASCII-hex `CRON_SECRET` values. If anyone ever sets `CRON_SECRET` to a value containing astral-plane chars (U+10000+), each is encoded as a surrogate pair and the comparison is still correct char-by-char, but "length" reflects code units not grapheme count. Documented in Plan Design §7. Acceptable.
- Suggested fix: None. Document CRON_SECRET as "ASCII hex only" in the ops doc.
