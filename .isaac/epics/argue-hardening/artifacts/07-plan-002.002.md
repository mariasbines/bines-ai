# Plan — Story 002.002

Admin view at `/argue/log`.

Server-rendered React Server Component. Reads the selected day's JSONL through the 002.001 storage
library. Scrubs harm-flagged entries by default via native `<details>`. Gated by Vercel Deployment
Protection at the project level (no app-level auth). Hidden from robots and unlisted in the sitemap.

## File-level intent

### CREATE `src/app/argue/log/page.tsx`

Async Server Component (Next 15 App Router conventions). No `'use client'`. No React state. No
custom fetching — just `await readArgueLogDay(day)` + `await listArgueLogDays()` from the 002.001
storage module.

Signature: `export default async function Page({ searchParams }: { searchParams: Promise<{ day?: string }> })`.
In Next 15, `searchParams` is a Promise and must be awaited (pattern used elsewhere in this codebase).

Responsibilities (AC by AC):
- **AC-001**: Async Server Component, no client JS.
- **AC-002**: Default to today's UTC day when `?day` absent. If `?day` present and matches the
  strict `YYYY-MM-DD` regex, use it. If present and doesn't match, render a polite error
  (`<p>not a valid day</p>`) + the day-index sidebar so navigation still works. Status is still
  200; no `notFound()` call.
- **AC-003**: Each entry rendered as a `<section>` with:
  - a `<header>` showing: timestamp (ISO), short IP hash (first 12 chars), turn count, verdict
    badge (colour-coded: ivory default, amber for `off_brand.length > 0`, ruby for
    `harm !== 'none'`), guard signals (if any), refused flag, model string, latencies
  - a `<div>` or `<ol>` rendering the turns sequentially (user / assistant), content in `<pre>`
    with `white-space: pre-wrap` — no HTML interpretation of user content (mitigates stored-XSS
    vector at the admin-view layer).
- **AC-004**: `verdict.harm !== 'none'` → entry wrapped in `<details>` without `open`, the turn
  content hidden behind a click. `<summary>` shows "hidden — flagged as: {harm}". Non-harm
  entries render with turns inline (no `<details>` wrapper).
- **AC-005**: Day navigation —
  - prev / next day links computed from `listArgueLogDays()` output (which is already
    desc-sorted). Prev = the day chronologically before the current one that has entries;
    next = the day after. If none, link is a disabled `<span>`.
  - a sidebar `<aside>` listing all days with entries (desc).
- **AC-006**: `export const metadata = { robots: { index: false, follow: false } }`. Also emit a
  `<meta name="robots" content="noindex, nofollow">` inside the component's returned JSX for
  belt-and-braces. Next's metadata export handles the `<head>`; the inline meta is redundant but
  harmless.
- **AC-007**: Grep verification — no link anywhere. Explicit: don't add `/argue/log` to
  `sitemap.ts`. Add an inline assertion in the route's test that the rendered page does not
  contain a link with `href="/argue/log"` (i.e. self-link check), and a separate test in the
  sitemap suite to verify the sitemap does not include the admin path.
- **AC-008**: No Blob URL in rendered HTML. The storage module returns parsed `ArgueLogEntry`
  objects — `b.url` from `list()` is consumed server-side inside `readArgueLogDay` and never
  returned. Page simply consumes entries; Blob URLs never appear. Test asserts the rendered
  HTML contains no `https://` substring that matches a Blob pattern.
- **AC-009**: Empty-day case — render "no conversations on this day" + the day navigation and
  index sidebar.
- **AC-010**: Storage errors — wrap the two storage calls in try/catch. On throw, render a
  minimal "log temporarily unreadable" message without leaking stack or env var names. Log the
  error via `console.error('[argue-log page] read failed')` with the error message only (no Blob
  URL, no token, no stack-trace dump).

## Risks to watch

- **Async searchParams shape drift** — Next 15 moved to Promise-shaped `searchParams`. Existing
  pages in this repo use the `Promise<{…}>` pattern. Verified by grepping `src/app/` for similar
  dynamic pages.
- **RSC + testing-library behaviour** — calling `await` on an async component and passing the
  result to `render()` is the established pattern in this codebase. Vitest + jsdom environment
  handles it.
- **Rendered HTML Blob URL inspection** — `readArgueLogDay` does NOT return URLs. Test asserts
  `container.innerHTML` doesn't contain `https://` AND doesn't contain `.blob.vercel-storage`.
  If either match ever lands, it's a real leak.
- **Test coverage of the no-open `<details>`** — `<details>` without `open` renders content in
  DOM but it's hidden from the user. RTL's `getByText` still finds it. Use
  `container.querySelector('details:not([open])')` instead for the structural assertion.
