## Design decisions

### 1 · Blob access mode — stay `public`, enforce URL-secrecy instead

`@vercel/blob` v2 requires `access: 'public'` on `put()`. There is no `access: 'private'` option at the SDK surface — the "private" posture in our architecture is achieved by **never exposing the returned URL to any client path**. We rely on URL unguessability plus the Deployment-Protection gate on `/argue/log` plus the no-public-linkage rule on the admin page.

AC-010 enforces this via a grep-audit test that asserts no blob URL substring appears in any `console.*` call or response body across the storage + cleanup modules.

### 2 · Storage shape — mirrors push-back exactly

Blob key: `argue-log/YYYY-MM-DD.jsonl`. One JSONL per UTC day, each entry one line. Same get-then-concat-then-put append pattern as `src/lib/push-back/storage.ts` with the same race tradeoff accepted for v1 (see risk ARGUE-OPS-003).

### 3 · Hash — Web Crypto SHA-256 of `salt + ip`

Pure function, no env reads inside. Caller (route in 002.004) reads `ARGUE_LOG_IP_SALT_CURRENT` and passes both args. This keeps the hash module trivially testable and keeps env-access concentrated at the route boundary.

```ts
const bytes = new TextEncoder().encode(salt + ip);
const digest = await crypto.subtle.digest('SHA-256', bytes);
return Array.from(new Uint8Array(digest))
  .map(b => b.toString(16).padStart(2, '0'))
  .join('');
```

Salt + ip concatenation order is locked (salt first). Documented so rotation doesn't accidentally invert.

### 4 · Delete API — list-then-del by URL

`@vercel/blob`'s `del()` accepts a blob URL. To delete a day we must:
1. `list({ prefix: 'argue-log/YYYY-MM-DD.jsonl', token })`
2. If blobs exist, call `del(blob.url, { token })` for each match (should be exactly one under v1 shape).
3. If no blobs match, return silently — idempotent delete.

This means delete internally surfaces the URL inside the function, but **never returns it, never logs it, never writes it to any response**. The grep audit catches accidental leaks.

### 5 · Schema — single file, one zod + types

`src/lib/argue-log/schema.ts` exports:
- `TURN` zod schema (`role` + `content`)
- `ARGUE_VERDICT` zod schema (harm enum, off_brand enum array, optional reasoning)
- `ARGUE_LOG_ENTRY` zod schema (schema_version: 1, timestamp, ip_hash (64 hex), salt_version, turns, guard_signals, verdict, refused, model, latency_ms)
- Inferred types: `Turn`, `ArgueVerdict`, `ArgueLogEntry`

Schema matches architecture §Components verbatim. Placing `ARGUE_VERDICT` here (not in `argue-filter/`) because (a) the log entry depends on it (the filter consumer reads it), (b) avoids a circular dep when 002.004's haiku.ts imports just the verdict schema.

### 6 · Cleanup route — GET with bearer auth, age filter, regex-validated keys

Vercel Cron invokes via GET with `Authorization: Bearer <CRON_SECRET>`. Route:
1. Read `process.env.CRON_SECRET`. If absent, return 500 (misconfiguration).
2. Extract `authorization` header. Missing or malformed → 401.
3. Timing-safe compare bearer token vs `CRON_SECRET`. Mismatch → 401.
4. List `argue-log/` prefix. Filter to keys matching `^argue-log/(\d{4}-\d{2}-\d{2})\.jsonl$` AND day strictly older than 90 days (UTC `now - 90 days`).
5. For each eligible key, `del()` with token.
6. Return 200 JSON `{ deleted: string[], skipped: number }` — only day-keys in the summary, never URLs.

Age boundary: "strictly older than 90 days" means a log from exactly 90 days ago is kept. The 91st day triggers deletion. Documented in AC-007 and unit test.

### 7 · Timing-safe comparison

No Node `crypto.timingSafeEqual` on edge runtime. Implement a 10-line equivalent using length-normalisation and XOR accumulation:

```ts
function timingSafeEqual(a: string, b: string): boolean {
  if (a.length !== b.length) return false;
  let diff = 0;
  for (let i = 0; i < a.length; i++) {
    diff |= a.charCodeAt(i) ^ b.charCodeAt(i);
  }
  return diff === 0;
}
```

Length-dependent early return is an acceptable leak (reveals token length, not content).

### 8 · `vercel.json` — create new, minimal

Does not exist yet. Create with only the cron schedule. If future stories need vercel config (headers, rewrites), they'll extend it.

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "crons": [
    { "path": "/api/argue-log/cleanup", "schedule": "30 3 * * *" }
  ]
}
```

03:30 UTC chosen for low-traffic window.

### 9 · Ops doc structure

Five sections per AC-009:
1. **Salt rotation (quarterly).** Three-step procedure: (a) generate new 32-byte hex via `openssl rand -hex 32`, (b) in Vercel env UI set `ARGUE_LOG_IP_SALT_PREVIOUS = <current value>` then `ARGUE_LOG_IP_SALT_CURRENT = <new value>`, (c) trigger a redeploy. No hash breakage mid-quarter.
2. **Cron secret rotation (annual).** Generate new token, set `CRON_SECRET`, redeploy. Vercel's cron invocation auto-picks up the new value.
3. **Manual cleanup sweep.** Single curl command with the current secret.
4. **Env-var setup.** Four-var checklist for first-time Production + Preview setup.
5. **Deployment Protection warning.** Never toggle off on Production.

### 10 · Partial log on client-abort — accept

Per Phase 6 open-item resolution: schema has no `truncated` field. If the tee catches partial assistant content before stream abort, that's what gets logged. Minimal behaviour.

import { z } from 'zod';

export const TURN = z.object({
  role: z.enum(['user', 'assistant']),
  content: z.string(),

export type Turn = z.infer<typeof TURN>;

export const ARGUE_VERDICT = z.object({

    ]),
  ),

});
export type ArgueVerdict = z.infer<typeof ARGUE_VERDICT>;

export const ARGUE_LOG_ENTRY = z.object({
  schema_version: z.literal(1),
  timestamp: z.string().datetime(),

  salt_version: z.enum(['current', 'previous']),

### Step 2 · Hash — `src/lib/argue-log/hash.ts`

```ts
export async function hashIp(ip: string, salt: string): Promise<string> {
  const bytes = new TextEncoder().encode(salt + ip);
  const digest = await crypto.subtle.digest('SHA-256', bytes);
  return Array.from(new Uint8Array(digest))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
}
```

No `server-only` import — the hash fn itself could in principle run in any Web Crypto environment, but in practice it's only called from the edge route. Keeping it free of a server-only gate simplifies tests.

### Step 4 · Storage — `src/lib/argue-log/storage.ts`

```ts
import 'server-only';
import { list, put, del } from '@vercel/blob';
import { ARGUE_LOG_ENTRY, type ArgueLogEntry } from './schema';

const PREFIX = 'argue-log/';
const DAY_KEY_RE = /^\d{4}-\d{2}-\d{2}$/;

function dayKeyUtc(d: Date = new Date()): string {
  return d.toISOString().slice(0, 10);
}

function filenameFor(day: string): string {
  return `${PREFIX}${day}.jsonl`;
}

function requireToken(): string {
  const token = process.env.BLOB_READ_WRITE_TOKEN;
  if (!token) throw new Error('BLOB_READ_WRITE_TOKEN missing');
  return token;
}

export interface AppendOptions {
  now?: Date;
}

export async function appendArgueLog(
  entry: ArgueLogEntry,
  options: AppendOptions = {},
): Promise<void> {
  // Schema-validate BEFORE hitting Blob. Reject malformed early.
  ARGUE_LOG_ENTRY.parse(entry);

  const token = requireToken();
  const filename = filenameFor(dayKeyUtc(options.now));

  const existing = await list({ prefix: filename, token });
  let body = JSON.stringify(entry) + '\n';
  if (existing.blobs.length > 0) {
    const res = await fetch(existing.blobs[0].url);
    if (res.ok) body = (await res.text()) + body;
  }

  await put(filename, body, {
    access: 'public',
    contentType: 'application/x-ndjson',
    addRandomSuffix: false,
    token,
    allowOverwrite: true,
  });
}

export async function listArgueLogDays(): Promise<string[]> {
  const token = requireToken();
  const out = await list({ prefix: PREFIX, token });
  const days = out.blobs
    .map(b => b.pathname.slice(PREFIX.length).replace(/\.jsonl$/, ''))
    .filter(isValidDayKey);
  return days.sort().reverse(); // newest first
}

export async function readArgueLogDay(day: string): Promise<ArgueLogEntry[]> {
  if (!isValidDayKey(day)) return [];
  const token = requireToken();
  const filename = filenameFor(day);
  const existing = await list({ prefix: filename, token });
  if (existing.blobs.length === 0) return [];
  const res = await fetch(existing.blobs[0].url);
  if (!res.ok) return [];
  const text = await res.text();
  const lines = text.split('\n').filter(l => l.length > 0);
  const entries: ArgueLogEntry[] = [];
  for (const line of lines) {
    const parsed = ARGUE_LOG_ENTRY.safeParse(JSON.parse(line));
    if (parsed.success) entries.push(parsed.data);
    // Silently skip malformed lines — schema drift recovery.
  }
  return entries;
}

export async function deleteArgueLogDay(day: string): Promise<void> {
  if (!isValidDayKey(day)) return; // invalid input → no-op, no throw
  const token = requireToken();
  const filename = filenameFor(day);
  const existing = await list({ prefix: filename, token });
  for (const blob of existing.blobs) {
    await del(blob.url, { token });
  }
}

function isValidDayKey(s: string): boolean {
  return DAY_KEY_RE.test(s);
}
```

Note: `@vercel/blob`'s `del()` signature verified against v2.3.3 — accepts URL or array of URLs.

### Step 5 · Cleanup route — `src/app/api/argue-log/cleanup/route.ts`

```ts
import { listArgueLogDays, deleteArgueLogDay } from '@/lib/argue-log/storage';

export const runtime = 'edge';

const GOVERNED_BY_HEADER = { 'X-Governed-By': 'bines.ai' } as const;
const RETENTION_DAYS = 90;

function timingSafeEqual(a: string, b: string): boolean {
  if (a.length !== b.length) return false;
  let diff = 0;
  for (let i = 0; i < a.length; i++) {
    diff |= a.charCodeAt(i) ^ b.charCodeAt(i);
  }
  return diff === 0;
}

function daysBetween(a: Date, b: Date): number {
  const ms = a.getTime() - b.getTime();
  return Math.floor(ms / (24 * 60 * 60 * 1000));
}

export async function GET(req: Request): Promise<Response> {
  const secret = process.env.CRON_SECRET;
  if (!secret) {
    console.error('[argue-log-cleanup] CRON_SECRET unconfigured');
    return new Response(JSON.stringify({ error: 'misconfigured' }), {
      status: 500,
      headers: { 'Content-Type': 'application/json', ...GOVERNED_BY_HEADER },
    });
  }

  const auth = req.headers.get('authorization') ?? '';
  const prefix = 'Bearer ';
  if (!auth.startsWith(prefix) || !timingSafeEqual(auth.slice(prefix.length), secret)) {
    return new Response(JSON.stringify({ error: 'unauthorized' }), {
      status: 401,
      headers: { 'Content-Type': 'application/json', ...GOVERNED_BY_HEADER },
    });
  }

  const now = new Date();
  const allDays = await listArgueLogDays();
  const deleted: string[] = [];
  let skipped = 0;

  for (const day of allDays) {
    const dayDate = new Date(`${day}T00:00:00Z`);
    if (Number.isNaN(dayDate.getTime())) {
      skipped++;
      continue;
    }
    if (daysBetween(now, dayDate) > RETENTION_DAYS) {
      await deleteArgueLogDay(day);
      deleted.push(day);
    } else {
      skipped++;
    }
  }

  return new Response(JSON.stringify({ deleted, skipped }), {
    status: 200,
    headers: { 'Content-Type': 'application/json', ...GOVERNED_BY_HEADER },
  });
}
```

## Test Strategy

**TDD discipline**: each test file written BEFORE its subject file. Red → green → minimal refactor. Expected test count: ~45-55 specs across four files.

**Coverage target**: ≥ 85% lines/functions/branches on `src/lib/argue-log/` and `src/app/api/argue-log/cleanup/` (above the 80% project floor; this is security-adjacent code).

**Edge cases required**:
- Time boundary: exactly 90.0 days old (not deleted), 90 + 1 second (deleted).
- Timezone: `dayKeyUtc` always UTC; test with a `now` Date in a different TZ to verify key still reflects UTC.
- Empty Blob: `list()` returns `{ blobs: [] }` → read returns `[]`, delete no-ops.
- Concurrency: NOT tested at integration level for v1 (race is accepted tradeoff). Unit test just verifies sequential behaviour.
- Token boundary: secret with unicode → timingSafeEqual still correct by `charCodeAt` (BMP only — CRON_SECRET will be ASCII hex, documented).

**Integration**: not required in this story. The storage functions will be exercised integration-style when 002.004 wires them into the chat route.

**Manual**: after merge to `dev`, confirm in Vercel dashboard that the cron schedule appears under Project → Settings → Crons, with a "next run" timestamp. No manual smoke-test needed on live Blob writes (the route that triggers writes doesn't exist yet).

## Rollback Plan

Each deliverable is additive — no existing file gets breaking changes:

- Revert: `git revert` the story's merge commit.
- New files (`src/lib/argue-log/`, `src/app/api/argue-log/cleanup/`, `docs/argue-log-ops.md`) simply disappear.
- `vercel.json` is new; revert removes it. Vercel falls back to default config.
- No env var changes required on Vercel for revert (the three new env vars are only *read* in 002.004 and the cleanup route; they can stay set harmlessly).
- No data migration. No Blob state exists yet that could be orphaned.

Revert is zero-risk. No downstream story (002.002, 002.004) has shipped yet, so nothing else breaks on revert.
