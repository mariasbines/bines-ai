# Story 002.002: Admin view at /argue/log

## Description

Maria's read-only view of the conversation log. A React Server Component at `/argue/log` that reads the selected day's JSONL via the storage library from 002.001, renders conversations, and scrubs harm-flagged content by default. Gated by Vercel Deployment Protection at the project level — no app-level auth, no public linkage.

Day-by-day navigation only. No cross-day loading, no client-side filtering, no infinite scroll. Server-rendered HTML plus minimal CSS. Matches the existing site's ivory/editorial-maximalist look without introducing a new design surface.

## Acceptance Criteria

- [ ] AC-001: `/argue/log` route exists at `src/app/argue/log/page.tsx` as a React Server Component (`async function Page`). No client-side JavaScript required for the core view.
- [ ] AC-002: Defaults to today's date when no query param present. `?day=YYYY-MM-DD` selects a specific day. Invalid date string renders a polite "not a valid day" server-rendered message (no 500).
- [ ] AC-003: Renders each log entry with: timestamp, short IP hash (first 12 chars of `ip_hash`), turn count, verdict badge, guard signals (if any), refused flag (if true), model used, latencies. Turns rendered sequentially (user / assistant), content in `<pre>` with `white-space: pre-wrap` to preserve line breaks without HTML interpretation.
- [ ] AC-004: Entries with `verdict.harm !== 'none'` render inside a native `<details>` element with content collapsed by default and the harm category visible as a summary badge (e.g., *"hidden — flagged as: hate"*). Entries with `harm === 'none'` render with content expanded (mitigates ARGUE-PRV-001 operator-wellbeing intent).
- [ ] AC-005: Day navigation: prev-day link, next-day link (disabled when the corresponding day has no entries), plus a sidebar or footer index listing all days with entries (from `listArgueLogDays()`, sorted newest first).
- [ ] AC-006: Page sets `export const metadata = { robots: { index: false, follow: false } }`. Also outputs `<meta name="robots" content="noindex, nofollow">` as a belt-and-braces.
- [ ] AC-007: No link to `/argue/log` from any other page on the site. `src/app/sitemap.ts` (or equivalent) does not include `/argue/log`. Verified by grep.
- [ ] AC-008: Page does not output any Blob URL anywhere in the rendered HTML. Storage reads happen server-side; only text content and metadata cross the server→client boundary. Verified by rendered-HTML inspection in test (mitigates ARGUE-SEC-001).
- [ ] AC-009: Empty-day case: when `readArgueLogDay(day)` returns `[]`, page renders a "no conversations on this day" message plus the day navigation (still functional).
- [ ] AC-010: Error case: if storage throws (e.g., Blob token misconfig), page renders a minimal server-side error without leaking stack traces or env-var names.

## Technical Tasks

1. Create `src/app/argue/log/page.tsx` — async Server Component, reads `searchParams.day` or defaults to UTC today, calls `readArgueLogDay()` and `listArgueLogDays()`, renders.
2. Small inline-scoped CSS (Tailwind v4 classes where possible, matches existing editorial-maximalist tokens). No new global styles.
3. Date helpers: `isValidDayKey(s: string): boolean` using the same regex from 002.001 (pull into a small shared helper if convenient — `src/lib/argue-log/day.ts`, optional).
4. Create `src/app/argue/log/__tests__/page.test.tsx` — uses Vitest + React testing approach consistent with project conventions. Mocks `src/lib/argue-log/storage.ts`. Covers: default-to-today, `?day` param, invalid date, harm-scrub-by-default, empty day, day navigation, noindex meta, no-URL-in-output.
5. Verify no link from home page, nav, or sitemap. Add assertion to a navigation test if one exists.

## Scope rules

- Do NOT add app-level auth. Deployment Protection is the gate (per ARGUE-SEC-005 decision).
- Do NOT add client-side JavaScript unless strictly needed. The view is server-rendered HTML.
- Do NOT add search, filter, or aggregation UI in v1. Future-nice, not needed for launch.
- Do NOT add a delete button or any mutation surface on this page. Read-only.
- Do NOT add MDX or Markdown rendering — log content is plain text, period (mitigates content-injection through the log view).
- Do NOT style with the SynapseDx palette. Inherit existing bines.ai ivory/editorial tokens.
