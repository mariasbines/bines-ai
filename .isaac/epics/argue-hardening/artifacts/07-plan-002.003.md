## File-level intent

### CREATE `src/lib/argue-filter/refusal.ts`

Exports:
- `REFUSAL_TEXT: string` — the locked text verbatim:
  `"not my lane — maria doesn't have a public position on this, and i don't invent them. what else have you got?"`
- `refusalEventStream(): ReadableStream<Uint8Array>` — returns a UTF-8-encoded `ReadableStream` that
  emits a single SSE body in the Anthropic SDK frame shape the existing `postChat` client already
  parses.

The client at `src/lib/chat/client.ts` looks for lines starting `data: ` and parses JSON; it pulls
`text` out of events matching `type === 'content_block_delta'` and `delta.type === 'text_delta'`. So
the body I emit must contain at least one such `data: ` line to surface the refusal text. A
`message_stop` sentinel after it ends the stream cleanly (client simply stops on `done`).

Frame design (matches Anthropic's SSE line-based shape — one blank line between events, each line
beginning `data: `):

```
data: {"type":"content_block_delta","index":0,"delta":{"type":"text_delta","text":"<REFUSAL_TEXT>"}}

data: {"type":"message_stop"}

```

The stream is built as a `ReadableStream` whose `start(controller)` enqueues one UTF-8-encoded
`Uint8Array` with the whole body, then closes. Single-chunk is fine — no artificial chunking, no
size-based splitting. The refusal is ~130 chars.

Why single chunk: simpler test, deterministic byte count, no race to flush. The existing client
handles partial lines correctly regardless (buffer until `\n`).

### MODIFY `src/lib/chat/system-prompt.ts`

Append a new section before the final `Now:` line:

```
# Out-of-scope topics

Some topics sit outside what this chat is here for. When a visitor brings one up, deflect in voice — don't engage substantively, don't lecture, don't apologise at length. A short redirect with a hook back to what's on the site. The shape: acknowledge it's not in-bounds, note there's no public position to draw on, hand the turn back.

Out-of-scope categories:
- electoral politics (parties, candidates, voting, elections)
- hot-button social issues (abortion, gun control, immigration policy, trans debate)
- race as identity politics (not the lived-experience side — that's fair game if it's on the site)
- religion (doctrine, practice, who's right)
- named real people outside what this site has published (colleagues, customers, competitors, public figures beyond passing reference)
- family of Maria beyond what's on the site (her son Zac, daughter Morgan, husband, sister/brother-in-law appear with specific details; invent nothing more)
- conspiracy theories and crypto hype (pump-and-dump culture, flat earth, etc.)

For any of the above: deflect briefly, in voice, and move on. Don't improvise a position Maria hasn't publicly taken.
```

Why this shape:
- Names each of the seven Q6 categories verbatim (AC-003, AC-004).
- Instructs *shape* of deflection not *exact string* — `REFUSAL_TEXT` stays the single source of
  truth (AC-003 bullet 3, scope rule "Do NOT include REFUSAL_TEXT verbatim").
- Slots neatly before the `Now:` closer so the existing "argue with visitors" instruction lands
  last.
- Content in-voice per existing antipattern rules — no "it is imperative", no legalese.

## Risks to watch

- **Stream shape drift from real SDK frames**. Mitigation: test parses the body with the same
  rules the client uses (`data: ` + JSON + `content_block_delta` with `text_delta`). If the
  Anthropic SDK emits a different frame shape, the client already handles unknown frame types
  via silent skip, so extra belt-frames don't break anything; the text-delta frame alone is the
  load-bearing assertion.
- **System-prompt section bloats**. The existing prompt is already ~4.5 KB. Adding ~1 KB pushes
  Sonnet's input prompt cost slightly. Per architecture doc, still well below Sonnet's 200k-token
  window. No mitigation needed in v1.
- **Refusal text escapes JSON**. The `content_block_delta` event embeds the refusal text inside a
  JSON string. Since `REFUSAL_TEXT` contains an apostrophe (`doesn't`, `don't`), standard
  `JSON.stringify` handles escaping correctly. Confirmed — no hand-escaping needed. Test asserts
  the decoded body parses back to the original text.
