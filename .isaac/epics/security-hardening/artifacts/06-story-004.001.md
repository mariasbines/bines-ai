# Story 004.001: Phase-2 hardening — CSP + admin noindex + judge-route rate-limit

## Description

Tightens the residual security surface left after the Phase-1 sweep
(`1cd4c53`). The Phase-1 work added security response headers, UA-based
bot-block middleware with a snarky 403, and an explicit per-User-Agent
allow/disallow `robots.txt`. This story closes the remaining gaps the
audit surfaced — items that were intentionally deferred because they
need careful handling, not because they're low priority.

Four deliverables:

1. **Content Security Policy (CSP) with nonces.** The biggest residual
   XSS-defence-in-depth lever. Done wrong, CSP breaks the site silently,
   so this needs nonce wiring through the React/Next runtime + the inline
   JSON-LD blocks + the Vercel Analytics injection. Probe paths after.
2. **`X-Robots-Tag: noindex` on admin-only routes.** `/argue/log` is
   already gated by Vercel Deployment Protection, but adding the
   `noindex` header is belt-and-braces — search engines never index the
   admin surface even if the protection layer ever misconfigures.
3. **Rate-limit on `/api/argue-judge/run`.** The new chat-end beacon
   endpoint added in pushback-v2 story 003.004 has no rate-limit. Without
   it, an attacker can replay the endpoint to burn judge-budget. Should
   reuse the Upstash pattern from `getChatRatelimit` but with its own
   key namespace.
4. **CI guard tests.** A vitest spec that GETs every public route in
   tests and asserts the security headers are present (HSTS, X-Frame,
   X-Content-Type, Referrer-Policy, Permissions-Policy, plus the new
   CSP). Catches drift if a future Next config change removes them.

## Acceptance Criteria

- [ ] **AC-001** Content Security Policy on every response. `Content-Security-Policy`
      header set via `next.config.ts` headers() OR via middleware. Directives:
      - `default-src 'self'`
      - `script-src 'self' 'nonce-<request-nonce>' https://va.vercel-scripts.com` (Vercel Analytics)
      - `style-src 'self' 'unsafe-inline'` (Tailwind + Fraunces/Inter inline-style requirements; tighten with nonces if achievable)
      - `img-src 'self' data: blob: https:` (poster frames, OG images)
      - `media-src 'self' blob:` (video posters, recantation video)
      - `font-src 'self' https://fonts.gstatic.com`
      - `connect-src 'self' https://api.anthropic.com https://*.upstash.io` (chat + rate-limit)
      - `frame-ancestors 'none'` (matches existing X-Frame-Options: DENY)
      - `form-action 'self'`
      - `base-uri 'self'`
      - `object-src 'none'`
      - `upgrade-insecure-requests`
- [ ] **AC-002** Per-request nonce minted in middleware (already running).
      Nonce attached to every inline `<script>` tag (currently: the JSON-LD
      blocks in `<JsonLd>`). Verify via curl that the nonce changes per
      request.
- [ ] **AC-003** `<JsonLd>` component reads the nonce from a request-scoped
      source (e.g. `next/headers` `headers()` in the server component) and
      attaches it to the `<script>` element.
- [ ] **AC-004** Browser console clean on every page after CSP ships (no
      CSP violations). Probe set: `/`, `/fieldwork`, `/fieldwork/06-brain-swap`,
      `/postcards`, `/postcards/001`, `/changed-my-mind`, `/changed-my-mind/04-singularity-different-clothes`,
      `/now`, `/taste`, `/argue`, `/argue` after a chat send, `/about`, `/privacy`,
      `/archive`. Document any violations in a short follow-up note.
- [ ] **AC-005** `/argue/log` and any future admin route emit
      `X-Robots-Tag: noindex, nofollow` in addition to existing Vercel
      Deployment Protection. Implemented in the route file (or a small
      shared helper) — NOT relying on robots.txt alone.
- [ ] **AC-006** `/api/argue-judge/run` is rate-limited per-IP. Reuse
      Upstash; recommend a separate ratelimit instance keyed on
      `judge:<ip>` with limit 30/hour (the endpoint should fire ~once per
      conversation; 30/hour is generous). Misconfigured Upstash env vars
      → permissive (matches existing chat/rate-limit posture for dev).
- [ ] **AC-007** New test file `src/__tests__/security-headers.test.ts`
      (or similar location) that boots the dev request handler and
      asserts each public route returns the full security-header set,
      including the new CSP. Acts as drift detection for future
      `next.config.ts` changes.
- [ ] **AC-008** No regression on the existing 552 tests. New tests pass.
      Coverage on changed modules ≥ 80%.

## Technical Tasks

1. Decide nonce strategy: middleware mints + injects, or per-route. Recommend
   middleware (already runs) using `crypto.randomUUID()` and forwarding via a
   request header that server components read.
2. Wire nonce into `<JsonLd>` (and any other inline-script component).
3. Construct CSP header string with the nonce interpolated. Choose between
   `next.config.ts` static headers (no nonce support there) vs middleware-based
   header (nonce-capable). Likely middleware.
4. Walk the probe set (AC-004) once locally; iterate CSP directives against
   any browser console violation.
5. Add `X-Robots-Tag: noindex, nofollow` to `/argue/log` route handler.
6. Add a new Upstash ratelimit instance + wire into `/api/argue-judge/run`.
   Mirror the `getChatRatelimit` shape.
7. Write the security-headers vitest spec.

## Scope rules

- Do NOT add CSP report-uri / report-to without a real reporting endpoint to
  receive violations. Skipping reporting is fine for v1; the browser console
  is the reporting channel.
- Do NOT block `style-src 'unsafe-inline'` if Tailwind / Next requires it —
  document the carve-out in the CSP comment block instead. Tightening style-src
  to nonces is a future optimisation.
- Do NOT add CSRF tokens — there's no auth-cookie surface for visitors and
  every state-changing API is JSON-content-type validated. CSRF is theatre here.
- Do NOT reach for a third-party WAF (Cloudflare etc) — Vercel platform handles
  the common DDoS / rate-flood patterns. Out of scope.
- Do NOT remove the existing `X-Frame-Options: DENY` even though
  `frame-ancestors 'none'` in CSP is the modern equivalent — older browsers
  honour the older header. Keep both.
