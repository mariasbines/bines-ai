# Story 003.004: /api/argue-judge/run route (Phase B)

**Epic:** `pushback-v2`
**Phase:** B — judge populates silently (no public render yet)

## Description

The visitor-triggered judge endpoint. Edge POST route at `/api/argue-judge/run` that receives a `{ conversation_id }` payload (sent via `navigator.sendBeacon` from the chat-end signal in story 003.005), validates IP-bind + recency + idempotency, fetches the conversation's turns from `argue-log`, runs `judgeConversation()` from the module shipped in 003.003, and writes the verdict to `argue-judges/`. **No shared secret** — the auth model is "your hashed IP must match the latest argue-log entry's hashed IP within 30 minutes" + "an existing verdict for this conversation_id makes the call a 200 no-op".

This route is the public-facing half of the judge. It ships in Phase B because it has no surface implications — judges accumulate silently in the Blob namespace during the soft-launch window between Phase B merge and Phase C ship.

## Acceptance Criteria

- [ ] AC-001: `src/app/api/argue-judge/run/route.ts` exists. `export const runtime = 'edge'`. POST handler. Request body schema (in-file Zod): `RUN_REQUEST = z.object({ conversation_id: z.string().uuid() })`. Malformed body → 400 + `{ category: 'validation' }`.
- [ ] AC-002: After validation, the route hashes the request IP using `hashIp()` from `src/lib/argue-log/hash.ts` with `ARGUE_LOG_IP_SALT_CURRENT`. Computes both `ipHashCurrent` AND `ipHashPrevious` (using `ARGUE_LOG_IP_SALT_PREVIOUS` if present) — salt-rotation tolerance pattern. If `ARGUE_LOG_IP_SALT_CURRENT` is unset, returns 500 + logs `[argue-judge] salt-unconfigured` (mirrors chat-route fail-loud).
- [ ] AC-003: Conversation lookup: `readArgueLogDay(today)` + `readArgueLogDay(yesterday)`, filter entries by `entry.conversation_id === body.conversation_id`. If empty → 404 + `{ category: 'not_found' }` (no information about whether the id ever existed).
- [ ] AC-004: Recency check: let `latest = entries.sort((a,b) => b.timestamp - a.timestamp)[0]`. If `(now - latest.timestamp) > 30 minutes` → 410 Gone + `{ category: 'expired' }`. Sweep cron (story 003.006) is the path for older conversations.
- [ ] AC-005: IP-bind check: if `latest.ip_hash !== ipHashCurrent && latest.ip_hash !== ipHashPrevious` → 403 + `{ category: 'forbidden' }`. Salt-rotation succeeds when the latest entry was written under PREVIOUS salt and now CURRENT is the active salt — same try-current-then-previous pattern as argue-log read paths.
- [ ] AC-006: Idempotency check: `existing = await findVerdictByConversationId(body.conversation_id)`. If `existing !== null` → 200 OK `{ judged: true, verdict: existing }`. No Sonnet call, no second write. (`sendBeacon` from the client gets a 200 always when the route is reachable; the visitor never sees the response — but downstream tests + admin tooling rely on the shape.)
- [ ] AC-007: Run path: assemble turns from ALL matching entries (sorted by `timestamp` ascending), call `judgeConversation(body.conversation_id, latest.from_slug ?? null, turns)`. On success → `appendArgueJudge(verdict)` → return 200 `{ judged: true, verdict }`. On `judgeConversation` throw → log `[argue-judge] run-failed: <reason>` and return 502 + `{ category: 'upstream' }`. NO partial verdict written.
- [ ] AC-008: Every response carries `X-Governed-By: bines.ai` header (matches existing argue-log + chat conventions). Tested on success AND every error path (400, 401-N/A here, 403, 404, 410, 500, 502).
- [ ] AC-009: 401 status code is **not used by this route**. The auth model has no shared secret to be missing — 401 is reserved for cron-secret routes (`/api/argue-log/cleanup`, the future `/api/argue-judge/sweep`). 404 covers "conversation_id unknown to us"; 403 covers "we know it, your IP doesn't match"; 410 covers "we know it, it's too old to judge from a browser request". Distinct codes prevent attacker oracle on which ids are real.
- [ ] AC-010: `src/app/api/argue-judge/run/__tests__/route.test.ts` covers full matrix:
  - Invalid body (missing/malformed `conversation_id`) → 400
  - Missing `ARGUE_LOG_IP_SALT_CURRENT` → 500
  - Unknown `conversation_id` (no entries match) → 404
  - Latest entry > 30 min old → 410
  - Latest entry IP_hash mismatch (under both salts) → 403
  - **Salt-rotation success**: latest entry written under PREVIOUS salt, CURRENT is now different → IP-bind passes via PREVIOUS hash → 200 + verdict written
  - Existing verdict (idempotent replay) → 200 + `{ judged: true, verdict: <existing> }` + NO call to `judgeConversation`
  - `judgeConversation` throws → 502 + log entry; no `appendArgueJudge` call
  - Success path → 200 + verdict; `appendArgueJudge` called exactly once with the returned verdict
  - `X-Governed-By: bines.ai` present on all of the above (success + error paths)
- [ ] AC-011: Quality gates green: `pnpm typecheck && pnpm lint && pnpm test`. Coverage on `src/app/api/argue-judge/run/route.ts` ≥85%.

## Scope rules

- Do NOT add a shared-secret bearer auth to this route. The auth model is IP-bind + recency + idempotency by architectural decision.
- Do NOT call `getJudgesForSlug` or any build-time loader from this route. Storage-write is one direction; loader-read is the other; never crossed.
- Do NOT add the chat-end signal in `<ChatInterface>`. Story 003.005.
- Do NOT add the sweep cron route. Story 003.006.
- Do NOT modify `vercel.json`. Story 003.006.
- Do NOT log visitor message content via `console.*`. Logs include `conversation_id`, error category, and salt-rotation hits — never typed text.
- Do NOT use 401 status. The route has no missing-secret failure mode.
- Do NOT add a retry on `judgeConversation` throw. Sweep cron picks the conversation up next 04:30 UTC.
- Do NOT use Node `crypto` or `fs`. Edge runtime requires Web Crypto + `@vercel/blob` SDK only.

## Risk mitigations (from 04-assessment.md)

- **PB2-SEC-002** `/api/argue-judge/run` IP-bind bypass → AC-005 (IP-hash match required), AC-002 (salt-rotation tolerance), AC-004 (30-min recency window), AC-006 (idempotency makes replay a no-op), AC-009 (404/403/410 distinct codes prevent oracle), AC-010 (full failure-mode test matrix)
- **PB2-SEC-007** beacon payload tampering → covered by PB2-SEC-002's IP-bind (a spoofed conversation_id from a different IP returns 403)
- **PB2-OPS-003** race between idle beacon, pagehide beacon, and sweep → AC-006 idempotency check returns 200 no-op on second fire
