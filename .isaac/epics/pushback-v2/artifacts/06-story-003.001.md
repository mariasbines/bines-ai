# Story 003.001: Schema diff + conversation_id threading (Phase A)

**Epic:** `pushback-v2`
**Phase:** A — schema + threading (no UI change, no judge yet)

## Description

Phase A foundation. Threads a stable `conversation_id` through the chat surface so every `argue-log` entry written from this point forward carries an id we can reassemble turns by, and adds the additive `from_slug` field to the schema (populated `null` by default in this story — Phase B story 003.002 wires actual capture). No judge module, no UI change, no public surface — purely the plumbing that everything else in the epic depends on.

After this story merges to `dev`, every new `argue-log` entry has `conversation_id` (UUID v4) and `from_slug` (always `null` until 003.002 lands). Old entries continue to parse without either field. Visitor experience is unchanged. This is safe to ship behind no flag.

Round-trip parse on a pre-Phase-A entry shape (no new fields) succeeds; round-trip on a new entry shape (both fields present) succeeds.
