# Story 003.007: <PushbackSummary> + build-time enrichment + count badge (Phase C)

**Epic:** `pushback-v2`
**Phase:** C ‚Äî public surfaces (release-blocker bundle starts here)

## Description

The first Phase C story. Brings the silent argue-judges corpus to the public site:

1. `getFieldworkBySlug` is extended to call `getJudgesForSlug` (shipped in 003.003) at build time and return a top-level `pushback: PushbackEnrichment` field on the `Fieldwork` shape. The static MDX `pushback.count` frontmatter is no longer authoritative; runtime reads only the derived shape.
2. `<PushbackSummary>` is a new server component rendered between `<MdxBody>` and `<FieldworkArticleFooter>` on `/fieldwork/[slug]`. Returns `null` when `count === 0` (silent absence). Renders header + count + landed line (if `landed > 0`) + top-3 verbatim excerpts as `<blockquote>` with `‚Äî anonymous` attribution. Plain-text rendered (no `dangerouslySetInnerHTML`); 240-char component-level truncation as belt-and-braces atop the schema gate.
3. `<FieldworkCard>` adds a small accent-tinted count badge next to the existing "in rotation" pill when `piece.pushback.count > 0`.

This story does NOT add the `[ argue with this ]` CTA to `<FieldworkCardCtas>` (story 003.008). It does NOT delete v1 surface (story 003.009). It does NOT update the privacy disclosure (story 003.010 ‚Äî release-blocker bundle close).

**Soft-launch gate**: this story merges to `dev` only after the 7-day soft-launch window described in the DAG file has elapsed. Maria signs off on Phase C readiness based on admin review of `argue-judges/*.jsonl` during that week.

- [ ] AC-002: `src/lib/content/fieldwork.ts` `getFieldworkBySlug(slug)` extended:
  ```ts
  const piece = /* existing readMdxFile + Zod validation */;
  if (!piece) return null;
  const enrichment = await getJudgesForSlug(slug).catch((err) => {
  ```
