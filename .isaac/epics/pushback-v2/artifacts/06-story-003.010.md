## Description

The release blocker. Five deliverables, all of which must be green before pushback-v2 ships to `master`:

1. **Privacy disclosure update** — the chat disclosure on `<ChatInterface>` is updated to mention public quoting via the locked draft text in PRD §"Privacy disclosure update text". This MUST land in the **same PR** as the Phase C public render (003.007 + 003.008 + 003.009). PB2-PRV-001 is a release blocker: no production window where excerpts ship without the updated disclosure. Since this story is the close of Phase C, the merge of master happens after this story's commit lands and Maria signs off.
2. **End-to-end voice-check sweep** — Maria runs through a fresh probe set against the integrated preview deploy: `[ argue with this ]` CTA from a real Fieldwork piece → conversation with the piece-aware preface → close tab → wait for verdict to populate via the run route → next deploy renders `<PushbackSummary>` with the excerpt. Voice + harm-gate behaviour spot-checked at every step.
3. **Ops doc + voice-check rubric** — `docs/argue-judge-ops.md` (created in 003.006) is reviewed by Maria end-to-end; gaps fixed in this story. `docs/argue-voice-check.md` (existing from argue-hardening) is extended with new sections for: judge prompt, system-prompt preface, `<PushbackSummary>` static labels, `[ argue with this ]` CTA, updated privacy disclosure.
4. **Launch-readiness checklist** — pre-merge gate: typecheck/lint/tests/build green, sweep cron visible in Vercel dashboard, both crons producing successful runs in the dashboard log, Vercel Deployment Protection still on for `/argue/log`, no env vars added to Vercel project settings (zero-new-env-var policy verified).
5. **Soft-launch sign-off** — Maria reviews `argue-judges/*.jsonl` admin output during the 7-day soft-launch window between Phase B end (003.006) and Phase C merge (this story). If no systemic harm-gate misses or voice regressions, Maria signs off and master merge proceeds. Any blocking issue surfaced during the window creates a follow-up story before this one closes.

This is the only story in Phase C that touches `<ChatInterface>` (for the disclosure text) — surgical change. The bulk of effort is verification and ops review, not code.
