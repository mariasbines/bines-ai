# Story 003.009: v1 surface deletion (Phase C)

**Epic:** `pushback-v2`
**Phase:** C — public surfaces (release-blocker bundle)

## Description

Removes the v1 pushback surface entirely. `<PushBackModal>`, `/api/push-back/route.ts`, any unused `src/lib/push-back/` helpers, and all their tests are deleted in this story. Per architecture §`v1 deletion timing`, deletion happens **last in this epic, not first** — a botched merge of v2 must not leave the project with neither v1 nor a working v2. By the time this story runs, Phase C public render (003.007) and CTA (003.008) are merged on dev and validated; v1 is fully unreachable from the UI; deletion is mechanical cleanup.

After this story, `grep -r 'PushBackModal\|/api/push-back\|src/lib/push-back' src/` returns zero hits. The repo is fully on v2.

## Acceptance Criteria

- [ ] AC-001: `src/components/PushBackModal.tsx` is deleted (`git rm`).
- [ ] AC-002: `src/components/__tests__/PushBackModal.test.tsx` is deleted (or any equivalent test path the modal had).
- [ ] AC-003: `src/app/api/push-back/route.ts` is deleted.
- [ ] AC-004: `src/app/api/push-back/__tests__/route.test.ts` is deleted (or equivalent).
- [ ] AC-005: `src/lib/push-back/` directory: audit. If any helper module is imported only by the deleted modal/route, delete it. If any helper is reused elsewhere (e.g. `src/lib/push-back/storage.ts` is the storage pattern reference for the argue-log work — it might be referenced in argue-log tests or docs), preserve it. Document the audit decision per file in the commit message under "v1 audit:".
- [ ] AC-006: Any imports of `<PushBackModal>` or `/api/push-back` from elsewhere in the codebase are removed. The most likely callsite is `<FieldworkCardCtas>` — already cleaned up in 003.008. Verify by grep audit (AC-008 below).
- [ ] AC-007: Any pushback-related frontmatter helpers that were specific to v1 (e.g. `pushback.count` increment helper, if one exists) are deleted. The `pushback.count` field itself stays in the frontmatter Zod schema for forward-compat per architecture §`getFieldworkBySlug` — but any code that **wrote** to it (manual or automated) is gone.
- [ ] AC-008: Grep audit (run as part of CI or as a unit-test assertion):
  - `grep -r 'PushBackModal' src/` → 0 hits
  - `grep -r 'from "@/app/api/push-back' src/` → 0 hits (no imports from the deleted route)
  - `grep -r '/api/push-back' src/` → 0 hits in source (test fixtures and snapshots that pre-dated v2 should also be 0; if any survive in non-test paths investigate)
  - `grep -r 'PushBackModalProps' src/` → 0 hits
- [ ] AC-009: Quality gates green: `pnpm typecheck && pnpm lint && pnpm test && pnpm build`. The build must succeed without the deleted modules — confirms there were no live imports we missed.
