# Story 001.010: Changed-my-mind first-class route

## Description

Build `/changed-my-mind/[slug]` — the first-class render for "changed my mind" pieces. These are rare (~1 per year expected), weighty, and get different production treatment than ordinary Fieldwork: full piece, own URL, filmed as well as written (when media is present), foregrounded in the archive. This story builds the route + renderer, even though v1 ships with zero such pieces — the mechanic must work by launch so if Maria ever does change her mind on a position, the site is ready.

## Acceptance Criteria

- [ ] AC-001: `/changed-my-mind/[slug]` dynamic route renders a changed-my-mind piece with `<ChangedMyMindArticle>` — visually distinct from `<FieldworkArticle>` (more prominent header, explicit "I changed my mind about →" preamble referencing the original Fieldwork slug)
- [ ] AC-002: Frontmatter schema extended: `supersedes: string` (slug of the original Fieldwork piece this supersedes — required); `originalPosition: string` (short summary of the old take); `newPosition: string` (short summary of the new take)
- [ ] AC-003: The referenced `supersedes` slug must exist as a retired Fieldwork piece — validated at build time
- [ ] AC-004: Archive page (from 001.009) `changed my mind` section links here, not to `/fieldwork/[slug]`
- [ ] AC-005: Page includes a visible *"read the original →"* link back to the original Fieldwork piece (still viewable with the retired banner)
- [ ] AC-006: Graceful if no changed-my-mind pieces exist: route still registered, static params returns empty array, no build errors
- [ ] AC-007: The change-my-mind piece gets its own per-page accent anchor — typically a rotating jewel or explicit in frontmatter

## Technical Tasks

1. Extend FieldworkFrontmatter schema to accept the new status `"changed-my-mind"` with the required `supersedes`/`originalPosition`/`newPosition` fields (likely as a discriminated union)
2. Build `src/app/changed-my-mind/[slug]/page.tsx` with `generateStaticParams`
3. Build `src/components/ChangedMyMindArticle.tsx` — variant of `FieldworkArticle` with the preamble and link back
4. Add validation in content loader: for status `"changed-my-mind"`, `supersedes` must reference a real Fieldwork slug
5. Update archive grouping (from 001.009) to link correctly

## Files

- **CREATE**: `src/app/changed-my-mind/[slug]/page.tsx`, `src/components/ChangedMyMindArticle.tsx`
- **MODIFY**: `src/lib/content/types.ts` (schema union), `src/lib/content/fieldwork.ts` (validation), `src/components/ArchiveSection.tsx` (linking)

## Scope rules

- Don't author a changed-my-mind piece in this story (v1 ships with zero — this is the mechanic, not the content)
- Don't build a diff-view between original and new (v2 consideration)

## Test plan (🟡 Critical Path)

- Unit: schema discriminated union accepts all four statuses, rejects invalid combinations
- Unit: cross-reference validation fails if `supersedes` points to non-existent slug
- Integration: route renders with a fixture changed-my-mind piece
- Integration: empty case (no such pieces) builds cleanly
