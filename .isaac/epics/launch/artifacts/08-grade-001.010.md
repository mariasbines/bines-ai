## Findings

### Medium (2)

**M-001 · Discriminated-union switch tightens schema for existing Fieldwork 01**

- Severity: MEDIUM
- File: `src/lib/content/types.ts` + `content/fieldwork/01-best-thing-not-at-work.mdx`
- Details: Fieldwork 01's frontmatter has `status: in-rotation` with no `supersedes/originalPosition/newPosition` — these three fields are dropped from the base schema and only required under the `changed-my-mind` variant. The `in-rotation` variant doesn't need them. **No changes to 001.005's content file required.** Verified.
- Impact: None (verified). Schema tightening is invisible to existing content.
- Suggested fix: None.

**M-002 · Cross-reference validation runs only at build time (not via unit test of the loader)**

- Severity: MEDIUM
- File: Plan Step 3 — `validateChangedMyMindReferences` is a separate export, not called inside `getAllFieldwork`
- Details: The validator is invoked explicitly from `/changed-my-mind/[slug]/generateStaticParams`. If a future loader caller forgets to call it, an invalid reference slips through. The alternative — call it inside `getAllFieldwork` — couples the loader to cross-reference invariants and breaks tests that use loader against partial fixtures.
- Impact: Low — for v1 there's only one caller that cares (`/changed-my-mind`). Adding the call in `generateStaticParams` catches it at build time, which is where we need it.
- Suggested fix: Consider moving to loader-level in v2 when there are more consumers. For v1, accept.

### Low (2)

**L-001 · `FieldworkStatus` type alias derivation from union discriminant**

- Severity: LOW
- File: Plan Step 1 — `export type FieldworkStatus = FieldworkFrontmatter['status']`
- Details: This works because TS extracts the union of `'in-rotation' | 'retired-still-right' | 'retired-evolved' | 'changed-my-mind'` automatically. No consumer-side change.
- Suggested fix: None.

**L-002 · Tests don't exercise the build-time-failure message from generateStaticParams**

- Severity: LOW
- File: Plan Step 6 test coverage
- Details: The unit tests exercise `validateChangedMyMindReferences` directly. The integration — that it actually runs at build and fails the build — isn't tested as an end-to-end.
- Suggested fix: Accept. Running `pnpm build` with a deliberately-broken fixture would be a manual test; not worth automating for v1.
