# Story 001.011: Chat API — streaming, rate limit, system prompt, agent-guard

## Description

The big one. Build the `/api/chat` streaming endpoint over the Anthropic API. This story carries the highest density of risk mitigations in the entire launch (9 risks from 04-assessment.md map here). Every AC is load-bearing — sizing this story generously is part of the risk strategy.

Includes: edge runtime route handler, Anthropic SDK client with streaming, voice-anchored system prompt, Moss-style agent-guard for injection detection, Upstash rate limit, input length/turn caps, error handling, and the *"AI not Maria"* guard rails in-prompt.

## Acceptance Criteria

- [ ] AC-001: `/api/chat` accepts POST with `{ messages: [{ role, content }] }`, returns a streaming response (edge runtime)
- [ ] AC-002: `src/lib/chat/system-prompt.ts` exports the system prompt. Prompt includes: (a) identity ("you are an AI trained to argue in Maria's voice — NOT Maria"), (b) voice rules (diagnostic not confessional, contrarian first, question hanging), (c) the antipatterns list as explicit refusal rules (no "7 things I learned", no "great question" openers, no corporate hedging), (d) Fieldwork 01 as a one-shot voice example (full text), (e) refusal rules for opinions Maria hasn't taken publicly, (f) safety refusals (no impersonation of specific real people beyond what the site publishes, no SynapseDx customer/deal claims, no medical/legal/financial advice), (g) AI-not-Maria cheerful acknowledgement when asked directly
- [ ] AC-003: Rate limit via `@upstash/ratelimit`: 10 messages per 10 min per IP baseline, 50 per day per IP hard cap, returns 429 with voice-appropriate message when exceeded (mitigates SEC-002)
- [ ] AC-004: Input length cap: reject messages > 4,000 chars with 413 (mitigates SEC-003)
- [ ] AC-005: Conversation history cap: last 10 turns passed to Anthropic; older truncated (mitigates SEC-003)
- [ ] AC-006: `max_tokens: 1024` on the Anthropic call (mitigates SEC-002 cost amplification)
- [ ] AC-007: Model: `claude-sonnet-4-6` (from `process.env.CHAT_MODEL` with default, env-tunable)
- [ ] AC-008: Agent-guard module (`src/lib/chat/agent-guard.ts`) detects: prompt-injection keywords (*"ignore previous"*, *"system prompt"*, *"reveal"*), role-swap attempts (*"pretend you are"*, *"act as"*, *"you are now"*), long single queries (>500 chars with zero conversation history). Detection LOGS signals and nudges the assistant via an appended system note. Does NOT hard-block (per mitigation SEC-001 — legitimate provocative queries must still get replies) (mitigates SEC-001)
- [ ] AC-009: API key read only server-side: `process.env.ANTHROPIC_API_KEY` referenced only in this route and `lib/chat/anthropic.ts`, never imported from client (mitigates SEC-005)
- [ ] AC-010: Error handling: distinguish upstream-error, rate-limited, and input-invalid in response codes and in server logs. No stack traces leak to client (mitigates OPS-001)
- [ ] AC-011: Response includes `X-Governed-By: bines.ai` header (adopted from Moss pattern)
- [ ] AC-012: Voice vibe-check procedure documented in `docs/chat-voice-check.md`: 20 sample prompts Maria can fire at the chat pre-launch, expected-voice rubric, feedback loop for prompt-tuning

## Technical Tasks

1. Build `src/lib/chat/anthropic.ts` — Anthropic client wrapper, reads key from env, throws friendly error if absent
2. Build `src/lib/chat/system-prompt.ts` — exports `SYSTEM_PROMPT` string composed from voice anchors + antipatterns + Fieldwork 01 one-shot (loaded via content pipeline)
3. Build `src/lib/chat/agent-guard.ts` — `detectAgentBehaviour(messages)` returns `{ isLikelyAgent, signals }` (follows the Moss pattern at `~/synapsedx-main/app/api/chat/agent-guard.ts`)
4. Build `src/lib/chat/rate-limit.ts` — Upstash ratelimiter factory reading `UPSTASH_REDIS_REST_URL/TOKEN`
5. Build `src/app/api/chat/route.ts` — edge runtime, streaming via `anthropic.messages.stream`, composes all of the above
6. Create `docs/chat-voice-check.md` — 20 sample prompts + expected register + Maria-facing rubric
7. Add Vitest tests for: prompt composition (voice anchors present); agent-guard detection (injection keywords, role-swap); input validation (length, turn cap); rate-limit invocation; error-path mapping

## Scope rules

- Do NOT implement the client-side chat UI. That's 001.012.
- Do NOT add conversation persistence — v1 is in-memory only.
- Do NOT add RAG, corpus-awareness, or vector search — that's v2.
- Do NOT add tools/function-calling — v1 is text-only.
- Do NOT log message content to server (privacy). Log signals and error categories only.
