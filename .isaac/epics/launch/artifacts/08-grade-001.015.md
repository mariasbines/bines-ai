### Medium (3)

**M-001 · Next 15 `params` is a Promise for dynamic route OG handlers too**

- Severity: MEDIUM
- File: Step 7 `src/app/fieldwork/[slug]/opengraph-image.tsx`
- Details: In Next 15, dynamic route `params` is a Promise that must be awaited. Plan's signature `async function Image({ params }: { params: { slug: string } })` should be `async function Image({ params }: { params: Promise<{ slug: string }> })` with `const { slug } = await params`.
- Impact: Build-time type error if not corrected.
- Suggested fix: Executor writes the awaited signature.

**M-002 · `getFieldworkBySlug` is server-only (fs) — can't run in edge OG handler**

- Severity: MEDIUM
- File: Step 7
- Details: Plan correctly omits `export const runtime` to let Next infer. The OG route should NOT be edge (since our content loaders are Node). Plan acknowledges this.
- Suggested fix: None. Omission of `runtime` is correct.

**M-003 · RSS body stripping — regex may miss nested/multiline JSX**

- Severity: MEDIUM
- File: Step 2 `to-html.ts` regex
- Details: `<PullQuote>multi-line content</PullQuote>` with the regex `<\/?([A-Z][A-Za-z0-9]*)[^>]*\/?>` strips only the tags but preserves inner text. Since Fieldwork 01's PullQuote has its content on a single line, this works. If future MDX has multi-line JSX component bodies with attributes that span lines, the regex could mis-strip.
- Suggested fix: Accept for v1. If future pieces use fancier JSX, upgrade to a proper MDX AST walk.
