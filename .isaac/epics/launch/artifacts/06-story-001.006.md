# Story 001.006: Postcard rendering — card + article + routes

## Description

Build the Postcards reading experience: `<PostcardCard>` for the short-form rendering, `<PostcardArticle>` for the single-postcard permalink, the `/postcards` index (stream view), and the `/postcards/[number]` dynamic route. Postcards are Chuck-Lorre-vanity-card style: numbered, brief, signed-off, no metadata strip. Content-forward.

## Acceptance Criteria

- [ ] AC-001: `<PostcardCard>` renders `POSTCARD #NNN`, short body (50-200 words), date line, signoff (`maria · dd mmm`). No metadata strip.
- [ ] AC-002: `<PostcardArticle>` is visually similar to the card but larger / prominent on its own page
- [ ] AC-003: `/postcards` renders all postcards in reverse-chronological order (newest first) with pagination ready but not required for v1 (fewer than 20 items)
- [ ] AC-004: `/postcards/[number]` dynamic route (e.g. `/postcards/001`) SSG-generated via `generateStaticParams` from `getAllPostcards()`
- [ ] AC-005: Nav link `Postcards` points to `/postcards` (already added in 001.002, now resolves)
- [ ] AC-006: Postcards use a different visual register than Fieldwork — no data strip, lighter weight. Per-card accent colour rotation keeps the jewels present but quieter.
- [ ] AC-007: Postcards are RSS-feed-eligible (001.015 will pick this up)

## Technical Tasks

## Scope rules

- Do NOT add comments, reactions, or any interaction to Postcards — they are read-only broadcasts
- Do NOT author content — Postcard #001 is seeded in 001.007
- Do NOT add a separate "tags" index page — tags on postcards are optional and not required in v1 navigation
