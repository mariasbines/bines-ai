# Implementation Plan: 001.010 — Changed-my-mind first-class route

## Overview

Build `/changed-my-mind/[slug]` — the first-class render for "changed my mind" pieces. V1 ships with zero such pieces; this story ensures the mechanic is ready when Maria ever does change her mind publicly. Distinct visual treatment (preamble referring to the superseded piece), cross-reference validation at build time, link back to the original.

🟡 Critical Path — breaking the mechanic would undermine the archive posture.

## Pre-Implementation Checklist

- [ ] HEAD `ad459d9` (001.009) or later
- [ ] Archive page linking correctly (001.009 already uses `linkPattern="changed-my-mind"` → `/changed-my-mind/[slug]`)
- [ ] Schema fields `supersedes`, `originalPosition`, `newPosition` already optional in `FieldworkFrontmatter` from 001.003
- [ ] No changed-my-mind pieces currently in `content/fieldwork/`

## Design decisions

### 1. Schema extension — discriminated union on status

Story AC-002: extend `FieldworkFrontmatter` into a discriminated union on `status` so that `status: "changed-my-mind"` REQUIRES `supersedes`, `originalPosition`, `newPosition`. Other statuses keep them optional.

**Approach:**

```typescript
// In types.ts — replace current flat schema with a discriminated union

const CHANGED_MY_MIND_EXTENSION = z.object({
  supersedes: z.string().min(1),        // required
  originalPosition: z.string().min(1),   // required
  newPosition: z.string().min(1),        // required
});

const BASE = z.object({ ...existing base fields except the three... });

export const FIELDWORK_FRONTMATTER = z.discriminatedUnion('status', [
  BASE.extend({ status: z.literal('in-rotation') }),
  BASE.extend({ status: z.literal('retired-still-right') }),
  BASE.extend({ status: z.literal('retired-evolved') }),
  BASE.extend({ status: z.literal('changed-my-mind') }).merge(CHANGED_MY_MIND_EXTENSION),
]);
```

This is the cleanest Zod discriminated union. The resulting TS type unions each variant — TypeScript narrows correctly based on `status`.

**BUT** — 001.003 already has the three fields as OPTIONAL top-level. Switching to a union is a schema-breaking change. We need to verify:

1. Does existing test `rejects invalid status enum` still pass? **Yes** — discriminated unions still enforce the tag field's enum.
2. Does existing test `accepts all four status values` still pass? **Yes** if we provide the required fields for `changed-my-mind`; the test currently uses the same base object for all four statuses, which will fail for `changed-my-mind` because the required fields are missing.

**Fix the existing test** to provide `supersedes/originalPosition/newPosition` for the changed-my-mind case — the test already loops over all four statuses with a spread-based construction, so we inject those three fields only for changed-my-mind.

### 2. Cross-reference validation

Story AC-003: `supersedes` must point at a real Fieldwork slug (retired-still-right or retired-evolved — the original piece that the new one supersedes). Enforce at **load time** via `getAllFieldwork`:

- After loading + validating all frontmatter, iterate `changed-my-mind` pieces and assert each `supersedes` exists in the loaded set AND has a retired status.
- Throw `ContentValidationError` with the offending file path + slug if mismatched.

Keep this check in a separate function (`validateChangedMyMindReferences`) so unit tests can exercise it against fixture data without needing the loader.

### 3. Route + static params

`/changed-my-mind/[slug]`:

- `generateStaticParams` reads `getFieldworkByStatus('changed-my-mind')` and emits one param per piece.
- Empty set = empty static params = build passes, route registers but produces nothing.

### 4. ChangedMyMindArticle component

Visually distinct from `<FieldworkArticle>`:
- Top label: `CHANGED MY MIND · superseding /fieldwork/[supersedes-slug]` in mono caps
- Preamble block: "I changed my mind about →" label + original-position summary + new-position summary
- MDX body below (reuse `<MdxBody>`)
- "Read the original →" link back to `/fieldwork/[supersedes-slug]`

Still uses the per-piece accent system.

### 5. Archive section update

001.009 already points changed-my-mind pieces at `/changed-my-mind/[slug]` via `linkPattern="changed-my-mind"`. No further edit needed. Plan earlier claims an update is required, but 001.009 pre-empted it.

### Step 1 · Extend Zod schema in `src/lib/content/types.ts`

Replace the flat `FIELDWORK_FRONTMATTER` with a discriminated union. Preserve all existing field definitions; only the outer structure changes.

```typescript
// Base fields common to all variants (everything except status + the changed-my-mind trio)
const FIELDWORK_BASE = z.object({
  id: z.number().int().positive(),
  slug: z.string().min(1).regex(/^[a-z0-9]+(-[a-z0-9]+)*$/, 'slug must be lowercase-kebab'),
  title: z.string().min(1),
  published: z.iso.date(),
  revised: z.array(z.iso.date()).optional(),
  retiredAt: z.iso.date().optional(),
  retiredReason: z.string().optional(),
  retiredReplacedBy: z.string().optional(),
  tags: z.array(TAG).min(1, 'fieldwork must have at least one tag'),
  media: MEDIA,
  pushback: PUSHBACK,
  changeMyMind: CHANGE_MY_MIND.optional(),
  excerpt: z.string().min(1),
  accent: ACCENT_TOKEN.optional(),
});

export const FIELDWORK_FRONTMATTER = z.discriminatedUnion('status', [
  FIELDWORK_BASE.extend({ status: z.literal('in-rotation') }),
  FIELDWORK_BASE.extend({ status: z.literal('retired-still-right') }),
  FIELDWORK_BASE.extend({ status: z.literal('retired-evolved') }),
  FIELDWORK_BASE.extend({
    status: z.literal('changed-my-mind'),
    supersedes: z.string().min(1),
    originalPosition: z.string().min(1),
    newPosition: z.string().min(1),
  }),
]);

export type FieldworkFrontmatter = z.infer<typeof FIELDWORK_FRONTMATTER>;
```

**Drop** the three `supersedes/originalPosition/newPosition` from the top-level flat schema (they now live only under the changed-my-mind variant).

**Also drop** the inline `z.enum(...)` assignment for `FIELDWORK_STATUS` — discriminated union replaces that enforcement but we still need the type alias:

```typescript
export type FieldworkStatus = FieldworkFrontmatter['status'];
```

### Step 3 · Add cross-reference validator

**File:** `src/lib/content/fieldwork.ts` (MODIFY)

Append:

```typescript
/**
 * Validate that each changed-my-mind piece's `supersedes` points at a
 * real Fieldwork slug with a retired status. Throws ContentValidationError
 * on mismatch (intended to run at build time from consumers like /archive
 * or /changed-my-mind generateStaticParams).
 */
export function validateChangedMyMindReferences(pieces: Fieldwork[]): void {
  const bySlug = new Map(pieces.map((p) => [p.frontmatter.slug, p]));
  for (const piece of pieces) {
    if (piece.frontmatter.status !== 'changed-my-mind') continue;
    const supersedes = piece.frontmatter.supersedes;
    const target = bySlug.get(supersedes);
    if (!target) {
      throw new ContentValidationError(
        piece.filePath,
        { supersedes },
        `${piece.filePath}: supersedes references unknown slug "${supersedes}"`,
      );
    }
    if (!(target.frontmatter.status === 'retired-still-right' || target.frontmatter.status === 'retired-evolved')) {
      throw new ContentValidationError(
        piece.filePath,
        { supersedes, targetStatus: target.frontmatter.status },
        `${piece.filePath}: supersedes target "${supersedes}" has status "${target.frontmatter.status}"; must be retired`,
      );
    }
  }
}
```

Re-export from barrel (`src/lib/content/index.ts`) — already wildcard-exports everything, no change needed.

### Step 5 · Route `/changed-my-mind/[slug]/page.tsx`

```typescript
import { notFound } from 'next/navigation';
import { getAllFieldwork, getFieldworkByStatus, validateChangedMyMindReferences } from '@/lib/content/fieldwork';
import { ChangedMyMindArticle } from '@/components/ChangedMyMindArticle';

export async function generateStaticParams() {
  const all = await getAllFieldwork();
  validateChangedMyMindReferences(all);  // build-time assertion (AC-003)
  const changed = all.filter((p) => p.frontmatter.status === 'changed-my-mind');
  return changed.map((p) => ({ slug: p.frontmatter.slug }));
}

interface PageProps {
  params: Promise<{ slug: string }>;
}

export default async function ChangedMyMindPage({ params }: PageProps) {
  const { slug } = await params;
  const all = await getAllFieldwork();
  const piece = all.find((p) => p.frontmatter.slug === slug && p.frontmatter.status === 'changed-my-mind');
  if (!piece) notFound();
  return <ChangedMyMindArticle piece={piece} />;
}
```

**Note**: the cross-reference validation lives in `generateStaticParams` so it runs at build time. An invalid reference fails the build with an informative error — exactly what AC-003 asks for.
