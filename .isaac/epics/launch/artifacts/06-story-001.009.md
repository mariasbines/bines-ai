# Story 001.009: Archive page тАФ organised by status

## Description

Build `/archive` тАФ the visible archive of retired Fieldwork pieces, organised not chronologically but by status: `still right` / `evolved ┬╖ refined` / `changed my mind (rare, first-class)`. The "changed my mind" bucket has special treatment (links to its own full route, treated as an event).

## Acceptance Criteria

- [ ] AC-001: `/archive` renders with three sections matching the spec: `still right (n)`, `evolved / refined (n)`, `changed my mind (n)` тАФ counts accurate
- [ ] AC-002: Each section lists its pieces with title + retirement date + one-line "why retired" note if present in frontmatter
- [ ] AC-003: "Changed my mind" pieces link to `/changed-my-mind/[slug]` (full route built in 001.010) тАФ not to the regular `/fieldwork/[slug]` route
- [ ] AC-004: Other retired pieces link to `/fieldwork/[slug]` (same route, just flagged retired)
- [ ] AC-005: Archive is empty-state aware: if no retired pieces exist yet, renders a graceful *"nothing retired yet тАФ come back"* message in Maria's voice
- [ ] AC-006: The `retired - still right` / `retired - evolved` status enum values from Zod schema are correctly grouped
- [ ] AC-007: Sort within each group: retirement date descending

## Technical Tasks
