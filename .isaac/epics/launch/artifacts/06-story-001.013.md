# Story 001.013: Video loop component + captions

## Description

Replace the `<VideoLoopPlaceholder>` from 001.004 with a real `<VideoLoop>` component that renders atmospheric AI-generated video loops in Fieldwork headers. Autoplay muted with captions, lazy-loaded for below-fold videos, `prefers-reduced-motion`-aware. Also supports the low-fi Maria-to-camera testimonial videos on the `[ watch ]` CTA.

## Acceptance Criteria

- [ ] AC-001: `<VideoLoop>` props: `src: string` (Blob URL), `captions?: string` (WebVTT Blob URL), `poster: string` (static poster frame URL), `alt: string`, `priority?: boolean` (first-above-fold vs lazy)
- [ ] AC-002: Autoplay + muted + loop + playsInline; poster frame shown first, video swaps in once ready
- [ ] AC-003: If `captions` provided, renders `<track kind="captions" src={captions} default>` — captions on by default
- [ ] AC-004: `prefers-reduced-motion: reduce` → autoplay disabled, static poster shown, subtle "▶ play" button to opt in manually
- [ ] AC-005: Lazy-load strategy: `priority` videos load eagerly; others use `loading="lazy"` on their poster image, video preloads `metadata` only until intersection-observer says it's near-viewport
- [ ] AC-006: Fieldwork header video pulled from frontmatter `media.headerVideo` + `media.captions` + `media.posterFrame` (all optional; absent = no video, just blank header or title on its own)
- [ ] AC-007: Testimonial `[ watch ]` CTA: if `media.testimonial` is present, clicking opens a dialog/modal with the testimonial video (also `<VideoLoop>` under the hood, captions on)
- [ ] AC-008: LCP target met: homepage < 2.0s, Fieldwork piece < 2.5s on throttled 4G (measured via Lighthouse)
- [ ] AC-009: Works without JavaScript (graceful degradation — native `<video>` tag with poster)

## Technical Tasks

1. Build `src/components/VideoLoop.tsx` (client component because `prefers-reduced-motion` and intersection observer need JS, but with no-JS fallback via `<noscript>` native video)
2. Build `src/components/WatchDialog.tsx` (client) — accessible modal for the testimonial videos
3. Update `src/components/FieldworkArticle.tsx` to replace `<VideoLoopPlaceholder>` with `<VideoLoop>`
4. Update `src/components/FieldworkCard.tsx` — hook up `[ watch ]` CTA to open `<WatchDialog>` when testimonial present (button disabled/hidden when absent)
5. Add frontmatter fields documented in 001.003's schema if they aren't already: `media.headerVideo`, `media.posterFrame`, `media.captions`, `media.testimonial`, `media.testimonialCaptions`
