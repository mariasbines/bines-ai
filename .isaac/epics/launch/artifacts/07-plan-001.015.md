## Design decisions

### 1. Combined RSS feed

One feed at `/rss.xml` containing Fieldwork + Postcards interleaved by `published` date. RSS 2.0 with `<content:encoded>` HTML. Use `feed` package.

For `<content:encoded>` we emit the raw MDX body as HTML — but our content is MDX not plain markdown. Strategy: strip MDX-specific JSX (like `<PullQuote>`) to leave prose, then pass through a markdown-to-HTML converter.

**Simpler approach:** use `marked` or `remark` to compile markdown (the non-JSX parts) and strip JSX component tags. Since `remark-gfm` is already installed, chain `remark → remark-gfm → remark-html`.

**Chosen:** `@mdx-js/mdx` already supports `evaluate` which returns a React component. For RSS we want HTML strings, not React. Use `remark` + `remark-html` + `remark-gfm`. These aren't installed — we'll add `remark` + `remark-html`.

Actually simpler: use a minimal manual pass. The body of Fieldwork pieces is mostly plain markdown with sparse `<PullQuote>` / `<Aside>` / `<FigureCaption>`. Feed readers don't render our custom components anyway. Options:

1. Add `remark` + `remark-html` → proper HTML conversion
2. Emit raw MDX body wrapped in `<pre>` (ugly in RSS readers)
3. Strip JSX tags with regex + compile rest via remark
4. Copy approach from `@mdx-js/mdx` evaluate → render to HTML string

**Chosen (pragmatic):** Option 3 — strip JSX via regex (the three component tags are finite and simple), then feed the remaining markdown through `remark` + `remark-html` for proper conversion. Install `remark` + `remark-html`.

### 2. Sitemap

Next 15 supports `app/sitemap.ts` exporting a default async function returning a `Sitemap` array. Dynamic entries derived from content loaders.

### 3. Robots

`app/robots.ts` returning a `{ rules, sitemap }` object. Disallow `/api/`, allow everything else.

### 4. Default OG

`public/og-default.png` — design via a simple `ImageResponse` invocation at `app/opengraph-image.tsx`. Generates 1200×630 at build time with the stamp + bio line + a jewel block.

### 5. Per-piece OG

`app/fieldwork/[slug]/opengraph-image.tsx` — `ImageResponse`-based per-slug OG with piece title + accent block + stamp.

Same pattern for `/postcards/[number]/opengraph-image.tsx` — simpler: `POSTCARD #NNN` + short excerpt + ivory ground.

### 6. Per-page metadata

Each page exports `generateMetadata`:
- `/` → title: 'bines.ai', description: bio
- `/fieldwork/[slug]` → title: piece.title, description: piece.excerpt
- `/postcards/[number]` → title: 'POSTCARD #NNN · bines.ai', description: short snippet
- `/archive`, `/now`, `/taste`, `/argue` → static metadata

Root layout already has `title: { default, template: '%s · bines.ai' }` — per-page metadata only needs to set `title` and the template applies.

### Step 2 · Small MDX→HTML helper

`src/lib/content/to-html.ts` — strip JSX component tags + compile remaining markdown to HTML.

```typescript
import 'server-only';
import { remark } from 'remark';
import remarkGfm from 'remark-gfm';
import remarkHtml from 'remark-html';

const JSX_TAG_PATTERN = /<\/?([A-Z][A-Za-z0-9]*)[^>]*\/?>/g;

export async function mdxBodyToHtml(body: string): Promise<string> {
  // Strip JSX custom components (keep their inner text)
  const stripped = body.replace(JSX_TAG_PATTERN, '');
  const result = await remark().use(remarkGfm).use(remarkHtml).process(stripped);
  return String(result);
}
```

**Note**: this is lossy for `<PullQuote>` — the italic emphasis is lost in RSS. Acceptable tradeoff for v1.

### Step 3 · `src/app/rss.xml/route.ts`

```typescript
import { Feed } from 'feed';
import { getAllFieldwork } from '@/lib/content/fieldwork';
import { getAllPostcards } from '@/lib/content/postcards';
import { mdxBodyToHtml } from '@/lib/content/to-html';

const SITE_URL = 'https://bines.ai';

export async function GET() {
  const feed = new Feed({
    title: 'bines.ai',
    description: 'Fieldwork and postcards from someone who actually uses the AI she writes about.',
    id: SITE_URL,
    link: SITE_URL,
    language: 'en',
    copyright: `© ${new Date().getFullYear()} Maria Bines`,
    feedLinks: {
      rss2: `${SITE_URL}/rss.xml`,
    },
  });

  const [fieldwork, postcards] = await Promise.all([
    getAllFieldwork(),
    getAllPostcards(),
  ]);

  const fwItems = await Promise.all(
    fieldwork.map(async (p) => ({
      kind: 'fieldwork' as const,
      slug: p.frontmatter.slug,
      title: p.frontmatter.title,
      date: new Date(`${p.frontmatter.published}T00:00:00Z`),
      description: p.frontmatter.excerpt,
      html: await mdxBodyToHtml(p.body),
      url: `${SITE_URL}/fieldwork/${p.frontmatter.slug}`,
    })),
  );
  const pcItems = await Promise.all(
    postcards.map(async (pc) => ({
      kind: 'postcard' as const,
      title: `Postcard #${pc.frontmatter.number.toString().padStart(3, '0')}`,
      date: new Date(`${pc.frontmatter.published}T00:00:00Z`),
      description: pc.body.slice(0, 200),
      html: await mdxBodyToHtml(pc.body),
      url: `${SITE_URL}/postcards/${pc.frontmatter.number.toString().padStart(3, '0')}`,
    })),
  );
  const items = [...fwItems, ...pcItems].sort((a, b) => b.date.getTime() - a.date.getTime());

  for (const item of items) {
    feed.addItem({
      title: item.title,
      id: item.url,
      link: item.url,
      date: item.date,
      description: item.description,
      content: item.html,
    });
  }

  return new Response(feed.rss2(), {
    headers: { 'Content-Type': 'application/rss+xml; charset=utf-8' },
  });
}
```

### Step 4 · `src/app/sitemap.ts`

```typescript
import type { MetadataRoute } from 'next';
import { getAllFieldwork } from '@/lib/content/fieldwork';
import { getAllPostcards } from '@/lib/content/postcards';

const SITE_URL = 'https://bines.ai';

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const [fieldwork, postcards] = await Promise.all([
    getAllFieldwork(),
    getAllPostcards(),
  ]);

  const staticRoutes: MetadataRoute.Sitemap = [
    { url: `${SITE_URL}/`, changeFrequency: 'weekly', priority: 1 },
    { url: `${SITE_URL}/postcards`, changeFrequency: 'weekly', priority: 0.8 },
    { url: `${SITE_URL}/archive`, changeFrequency: 'monthly', priority: 0.6 },
    { url: `${SITE_URL}/now`, changeFrequency: 'monthly', priority: 0.6 },
    { url: `${SITE_URL}/taste`, changeFrequency: 'monthly', priority: 0.5 },
    { url: `${SITE_URL}/argue`, changeFrequency: 'yearly', priority: 0.5 },
  ];

  const fwUrls: MetadataRoute.Sitemap = fieldwork.map((p) => ({
    url: `${SITE_URL}/fieldwork/${p.frontmatter.slug}`,
    lastModified: new Date(`${p.frontmatter.published}T00:00:00Z`),
    changeFrequency: 'monthly',
    priority: 0.8,
  }));

  const cmmUrls: MetadataRoute.Sitemap = fieldwork
    .filter((p) => p.frontmatter.status === 'changed-my-mind')
    .map((p) => ({
      url: `${SITE_URL}/changed-my-mind/${p.frontmatter.slug}`,
      lastModified: new Date(`${p.frontmatter.published}T00:00:00Z`),
      changeFrequency: 'yearly',
      priority: 0.7,
    }));

  const pcUrls: MetadataRoute.Sitemap = postcards.map((pc) => ({
    url: `${SITE_URL}/postcards/${pc.frontmatter.number.toString().padStart(3, '0')}`,
    lastModified: new Date(`${pc.frontmatter.published}T00:00:00Z`),
    changeFrequency: 'yearly',
    priority: 0.6,
  }));

  return [...staticRoutes, ...fwUrls, ...cmmUrls, ...pcUrls];
}
```

### Step 6 · Default OG image via `app/opengraph-image.tsx`

Next 15 conventions: `app/opengraph-image.{tsx,jpg,png}` produces the site-wide default at `/opengraph-image`. Using `ImageResponse`:

```typescript
import { ImageResponse } from 'next/og';

export const runtime = 'edge';
export const size = { width: 1200, height: 630 };
export const contentType = 'image/png';

export default function Image() {
  return new ImageResponse(
    (
      <div style={{ /* ivory ground, stamp-style type, bio line */ }}>
        {/* layout */}
      </div>
    ),
    { ...size },
  );
}
```

`ImageResponse` doesn't support Tailwind — it uses inline CSS with a subset of the `style` prop. Fonts are loaded via `ImageResponse`'s `fonts` option; for simplicity we use the default system fonts (acceptable for v1 OG cards).

### Step 7 · Per-Fieldwork OG

`src/app/fieldwork/[slug]/opengraph-image.tsx`:

```typescript
import { ImageResponse } from 'next/og';
import { notFound } from 'next/navigation';
import { getFieldworkBySlug } from '@/lib/content/fieldwork';
import { accentFor } from '@/lib/design/accent';

export const runtime = 'edge';  // but our loaders are node-server-only.
```

**Problem:** `/opengraph-image.tsx` generates at build time (SSG) per `generateStaticParams`. Since our loaders use `fs.readFile`, they need Node runtime. Next 15 lets us NOT specify runtime; by default these OG routes run on the same runtime as the page. Page is SSG'd, so OG generation happens at build time in the Node build process — fine.

**Chosen:** omit `export const runtime`; let Next choose (Node for SSG build).

```typescript
import { ImageResponse } from 'next/og';
import { notFound } from 'next/navigation';
import { getFieldworkBySlug } from '@/lib/content/fieldwork';
import { accentFor } from '@/lib/design/accent';

export const size = { width: 1200, height: 630 };
export const contentType = 'image/png';

const JEWEL_HEX: Record<string, string> = {
  emerald: '#0F7B5A',
  sapphire: '#1B3A8C',
  ruby: '#B0213A',
  topaz: '#C28F2A',
  amethyst: '#3F1E4C',
};

export default async function Image({ params }: { params: { slug: string } }) {
  const piece = await getFieldworkBySlug(params.slug);
  if (!piece) notFound();
  const accent = JEWEL_HEX[accentFor(piece)];
  const idPadded = piece.frontmatter.id.toString().padStart(2, '0');

  return new ImageResponse(
    (
      <div style={{
        width: '100%', height: '100%', background: '#FFFFF4', color: '#1A1814',
        display: 'flex', flexDirection: 'column', justifyContent: 'space-between', padding: 64,
      }}>
        <div style={{ fontSize: 24, textTransform: 'uppercase', letterSpacing: 4, color: accent }}>
          FIELDWORK {idPadded} · bines.ai
        </div>
        <div style={{ fontSize: 64, fontWeight: 900, lineHeight: 1.05, maxWidth: 960 }}>
          {piece.frontmatter.title}
        </div>
        <div style={{ fontSize: 20, color: '#1A181499' }}>
          {piece.frontmatter.excerpt.slice(0, 140)}
        </div>
      </div>
    ),
    { ...size },
  );
}
```
