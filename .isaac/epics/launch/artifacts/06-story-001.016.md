# Story 001.016: Vercel project setup + env vars + preview deploy

## Description

Create the Vercel project, link it to the GitHub repo, configure env vars for all upstream services (Anthropic, Vercel Blob, Upstash), and verify a preview deploy from the `dev` branch. The site runs end-to-end on a Vercel preview URL before domain cutover.

Heavy on Maria-executed actions in the Vercel dashboard — the story documents each step; Claude cannot do most of these without dashboard access.

## Acceptance Criteria

- [ ] AC-001: Vercel project `bines-ai` created, linked to `mariasbines/bines-ai` on GitHub
- [ ] AC-002: Framework preset auto-detected as Next.js; install command `pnpm install`; build command `pnpm build`; output `.next`
- [ ] AC-003: Production branch = `master`; `dev` branch exists and is configured as a preview branch
- [ ] AC-004: All required env vars set in Vercel dashboard (production + preview scopes as appropriate):
  - `ANTHROPIC_API_KEY`
  - `BLOB_READ_WRITE_TOKEN`
  - `UPSTASH_REDIS_REST_URL`
  - `UPSTASH_REDIS_REST_TOKEN`
  - `CHAT_MODEL` (optional, defaults to `claude-sonnet-4-6` in code)
  - `RATE_LIMIT_PER_WINDOW` (optional)
- [ ] AC-005: Vercel Blob store created on Maria's account; read/write token minted and set
- [ ] AC-006: Upstash Redis instance created on free tier; REST URL + token set
- [ ] AC-007: Anthropic API key minted with billing alert at a sensible threshold (Maria to set — suggested £50/month)
- [ ] AC-008: First preview deploy from `dev` branch succeeds; site reachable at Vercel preview URL
- [ ] AC-009: Chat tested against live Vercel preview — responses stream, rate limits work, voice sounds right
- [ ] AC-010: Push-back tested against live preview — submission lands in Blob
