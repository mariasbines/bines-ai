### High (1)

**H-001 · `usePrefersReducedMotion` named as a hook but called outside React render — rules-of-hooks violation**

- Severity: HIGH
- File: `src/components/ChatInterface.tsx` (proposed Step 5)
- Details: Plan defines `usePrefersReducedMotion` (hook naming) but calls it inside `handleSubmit` (an event handler, not a render). This violates React's rules-of-hooks and would trigger ESLint errors. Plan text acknowledges this: "Rename to `checkReducedMotion` (just a function). Executor fixes during write."
- Impact: If the executor overlooks the fix, `pnpm lint` will fail.
- Suggested fix: Rename to `checkReducedMotion` (regular function) OR move the check into a `useEffect` that stores the value in state. Plan already recommends the rename.

### Medium (2)

**M-001 · SSE frame parsing assumes Anthropic's exact stream shape**

- Severity: MEDIUM
- File: Plan Step 2 `client.ts` — `evt?.type === 'content_block_delta'`
- Details: The Anthropic SDK's `toReadableStream()` wraps stream events in a specific JSON shape. Plan assumes `{ type: 'content_block_delta', delta: { type: 'text_delta', text } }`. This matches Anthropic's documented stream protocol. If the SDK shifts (minor version bumps), parsing could break.
- Impact: Low — Anthropic's stream protocol has been stable. Mitigation: silently skip malformed frames (plan already does this via try/catch).
- Suggested fix: Accept. Add a comment in `client.ts` noting the Anthropic-SDK dependency and version tolerance.

**M-002 · `crypto.randomUUID()` availability in all target browsers**

- Severity: MEDIUM
- File: `ChatInterface.tsx`
- Details: `crypto.randomUUID()` requires secure context (HTTPS or localhost) and modern browsers. bines.ai production is HTTPS-only (apex domain). Local dev: localhost counts as secure context. Safe.
- Suggested fix: None. Works in production + dev.

### Low (3)

**L-001 · `ChatMessage` is a server component but imported from a client component chain**

- Severity: LOW
- File: `ChatMessage.tsx` (marked server by convention since no `'use client'`)
- Details: `<ChatMessage>` is stateless and could be client-bundled via its import in `<ChatInterface>` (a client component). Next 15 auto-bundles unmarked components as client when they're inside a client tree. This is correct behaviour — no action needed.
- Suggested fix: None.

**L-002 · Abort removes empty assistant placeholder, but abort mid-stream leaves partial content in state**

- Severity: LOW
- File: Step 5 handleAbort
- Details: If user aborts mid-stream, the partial assistant message stays in messages[]. The placeholder-filter only triggers on error paths. Aborted-then-resubmit includes the partial reply in the next turn's history. Probably fine — it's a real message the user saw.
- Suggested fix: Accept for v1.

**L-003 · No test for the aborted-streaming path**

- Severity: LOW
- File: Test plan
- Details: Aborted streams (Escape key) are tested in ChatInput but not the full ChatInterface integration. Acceptable for v1.
- Suggested fix: Accept.
