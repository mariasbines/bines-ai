## Design decisions

### 1. Edge runtime

```typescript
export const runtime = 'edge';
```

Next 15 edge runtime is canonical for streaming Anthropic responses. Note: `fs.readFile` is NOT available on edge. The system prompt's Fieldwork 01 one-shot must be loaded differently.

**Approach**: read Fieldwork 01 at build time via import-from-content-module or a build-time constant. Since our content loaders from 001.003 use `fs.readFile` (server-only, Node), they can't run on edge. We need a build-time load.

**Chosen:** load Fieldwork 01 body at MODULE-LOAD time via a server-component dynamic import or a build-time constant. Next's edge runtime supports `import.meta`-style build-time statics, but not filesystem reads. We'll:

1. Author a small helper `src/lib/chat/fieldwork-01.ts` that exports the body as a constant string (actually loaded via `import mdx from '../../../content/fieldwork/01-best-thing-not-at-work.mdx?raw'` — Next supports `?raw` imports in development, but not universally). 

**Simpler, more portable:** copy the body into the source as a TypeScript constant. The piece is the voice one-shot; coupling it to the chat prompt at build is desirable so the prompt doesn't drift from the content.

**Trade-off:** if Maria edits the MDX body post-launch without updating `fieldwork-01.ts`, the chat one-shot goes stale. Document this tradeoff in the source comment and add an item to the launch QA (001.018) to check parity.

Signals extended to cover bines.ai-specific attacks:
- `structured_extraction` — "list all", "enumerate", "json format"
- `prompt_injection_attempt` — "ignore previous", "system prompt", "instructions"
- `role_swap_attempt` — "pretend you", "act as", "you are now"
- `long_single_query` — first message >500 chars
- `system_reveal_attempt` — "reveal", "show me your prompt", "print your rules"

**Detect-and-log, not hard-block.** Per SEC-001 mitigation, the signals are appended as a system note before the assistant reply — they don't refuse to respond.

### 4. Rate limit module

```typescript
// src/lib/chat/rate-limit.ts
import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';

let _ratelimit: Ratelimit | null = null;

export function getChatRatelimit(): Ratelimit | null {
  if (_ratelimit) return _ratelimit;
  const url = process.env.UPSTASH_REDIS_REST_URL;
  const token = process.env.UPSTASH_REDIS_REST_TOKEN;
  if (!url || !token) return null; // Rate-limiting silently disabled if not configured
  const redis = new Redis({ url, token });
  _ratelimit = new Ratelimit({
    redis,
    limiter: Ratelimit.slidingWindow(10, '10 m'), // AC-003 baseline
    analytics: false,
    prefix: 'bines:chat',
  });
  return _ratelimit;
}

// Hard cap: 50/day/IP — separate Ratelimit instance
export function getChatDailyRatelimit(): Ratelimit | null {
  // similar pattern with '1 d' window
}
```

Returns `null` when env vars absent — in that case the route bypasses rate limiting (not ideal, but prevents CI failures and lets dev runs without Upstash config succeed). Production MUST set env vars; Maria configures during Vercel setup (001.016).

### 6. Input validation shape

```typescript
interface ChatRequest {
  messages: Array<{ role: 'user' | 'assistant'; content: string }>;
}

function validate(body: unknown): { ok: true; data: ChatRequest } | { ok: false; code: number; error: string } {
  // Body must be { messages: Array<...> }
  // Each message.content must be a string
  // Each message.content length <= MAX_MESSAGE_CHARS (413 if over)
  // messages.length must be >= 1
  // Truncate to last MAX_TURNS messages (not an error — silent truncation)
  // Returns validation result or error shape
}
```

Prefer Zod for validation consistency with the rest of the codebase.

### 7. Error handling

- `413` — input over length (caught in validate)
- `400` — malformed body
- `429` — rate-limited
- `500` — upstream Anthropic failure or unknown error
- No stack traces leak to client (log server-side only)
- Response shape: `{ error: string; category: 'rate-limited' | 'input' | 'upstream' | 'unknown' }`

### Step 6 · `src/lib/chat/validate.ts` (extracted from route for testability)

```typescript
import { z } from 'zod';

export const CHAT_REQUEST = z.object({
  messages: z.array(
    z.object({
      role: z.enum(['user', 'assistant']),
      content: z.string().min(1),
    }),
  ).min(1),
});

export const MAX_MESSAGE_CHARS = 4000;
export const MAX_TURNS = 10;

export type ValidateResult =
  | { ok: true; messages: Array<{ role: 'user' | 'assistant'; content: string }> }
  | { ok: false; status: 413 | 400; error: string; category: 'input' };

export function validateRequest(body: unknown): ValidateResult {
  const parsed = CHAT_REQUEST.safeParse(body);
  if (!parsed.success) {
    return { ok: false, status: 400, error: 'Invalid request shape', category: 'input' };
  }
  for (const m of parsed.data.messages) {
    if (m.content.length > MAX_MESSAGE_CHARS) {
      return {
        ok: false,
        status: 413,
        error: `Message too long (max ${MAX_MESSAGE_CHARS} chars)`,
        category: 'input',
      };
    }
  }
  const truncated = parsed.data.messages.slice(-MAX_TURNS);
  return { ok: true, messages: truncated };
}
```

### Step 7 · `src/app/api/chat/route.ts`

Orchestrates everything:

```typescript
import { getAnthropicClient, DEFAULT_MODEL, MAX_TOKENS } from '@/lib/chat/anthropic';
import { SYSTEM_PROMPT } from '@/lib/chat/system-prompt';
import { detectAgentBehaviour } from '@/lib/chat/agent-guard';
import { getChatRatelimit, getChatDailyRatelimit } from '@/lib/chat/rate-limit';
import { validateRequest } from '@/lib/chat/validate';

export const runtime = 'edge';

function jsonError(status: number, error: string, category: string) {
  return new Response(JSON.stringify({ error, category }), {
    status,
    headers: { 'Content-Type': 'application/json', 'X-Governed-By': 'bines.ai' },
  });
}

function getIp(req: Request): string {
  const xf = req.headers.get('x-forwarded-for');
  if (xf) return xf.split(',')[0].trim();
  return 'unknown';
}

export async function POST(req: Request) {
  let body: unknown;
  try {
    body = await req.json();
  } catch {
    return jsonError(400, 'Invalid JSON body', 'input');
  }

  const v = validateRequest(body);
  if (!v.ok) return jsonError(v.status, v.error, v.category);

  const ip = getIp(req);

  // Rate limit (two checks): 10/10min baseline + 50/day hard cap
  const baseline = getChatRatelimit();
  const daily = getChatDailyRatelimit();
  if (baseline) {
    const { success } = await baseline.limit(ip);
    if (!success) {
      return jsonError(
        429,
        'ease up — we\'ve had a lot of arguing today',
        'rate-limited',
      );
    }
  }
  if (daily) {
    const { success } = await daily.limit(`day:${ip}`);
    if (!success) {
      return jsonError(
        429,
        'that\'s your 50 for today. come back tomorrow',
        'rate-limited',
      );
    }
  }

  // Agent-guard — detect-and-log, not block
  const guard = detectAgentBehaviour(v.messages);
  if (guard.isLikelyAgent && process.env.NODE_ENV !== 'test') {
    // Log signals without logging message content (privacy)
    console.log(`[chat] agent-likely signals: ${guard.signals.join(',')}`);
  }

  let client;
  try {
    client = getAnthropicClient();
  } catch {
    return jsonError(500, 'chat is temporarily unavailable', 'upstream');
  }

  try {
    const stream = client.messages.stream({
      model: DEFAULT_MODEL,
      system: SYSTEM_PROMPT,
      messages: v.messages,
      max_tokens: MAX_TOKENS,
    });

    return new Response(stream.toReadableStream(), {
      headers: {
        'Content-Type': 'text/event-stream',
        'X-Governed-By': 'bines.ai',
      },
    });
  } catch (err) {
    console.error('[chat] anthropic stream error:', err instanceof Error ? err.message : 'unknown');
    return jsonError(500, "Claude's having a moment, try again in a sec", 'upstream');
  }
}
```
