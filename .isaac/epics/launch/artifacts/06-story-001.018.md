# Story 001.018: Launch QA — exit gate

## Description

The launch exit gate. Run through a pre-launch checklist covering performance, accessibility, security, voice, privacy, and operational readiness. Any failure here is a blocker; fixes get their own follow-up stories (or are inlined in this story if trivial). This story is what turns *"the site builds and deploys"* into *"the site is ready for humans."*

## Acceptance Criteria

- [ ] AC-001: **Core Web Vitals green** on homepage, a Fieldwork piece, a Postcard, `/now`, `/taste`, `/archive`, `/argue` — LCP < 2.5s, CLS < 0.1, INP < 200ms (Lighthouse mobile + desktop)
- [ ] AC-002: **Accessibility audit passes**: Lighthouse a11y score ≥ 95 on all pages; manual keyboard-nav pass through the whole site; screen reader announces chat responses correctly; captions present on any shipped videos; `prefers-reduced-motion` respected
- [ ] AC-003: **Chat voice vibe-check passes**: Maria runs 20 prompts from `docs/chat-voice-check.md` against production; each response passes the rubric (diagnostic not confessional, contrarian-first register, proper refusals for unknown opinions, AI-not-Maria disclosure when asked) — at least 18/20 pass on first run; failures feed a prompt-tuning commit if needed (BRD-001)
- [ ] AC-004: **Security pre-flight**:
  - `grep -r NEXT_PUBLIC src/` returns no entries referencing API keys, tokens, or secrets (SEC-005)
  - Prompt-injection spot-check: 5 known patterns tried against live chat; none leak system prompt, none break the persona (SEC-001)
  - Rate limit tested: script fires 15 messages in 30s → first 10 succeed, rest 429 (SEC-002)
  - Push-back honeypot tested: bot-style submission rejected (SEC-006)
  - Output plain-text check: ask chat for HTML — response is not rendered as HTML in the UI (SEC-004)
- [ ] AC-005: **Content QA**: Fieldwork 01 renders correctly — title, metadata, body, pull-quote, signoff; Postcard #001 renders correctly; antipatterns strip matches locked copy exactly; bio line matches locked copy; stamp visible as favicon and in header
- [ ] AC-006: **Privacy & legal**:
  - Push-back modal shows the privacy note verbatim (PRV-001)
  - Chat page shows the "sent to Anthropic / not stored" note (PRV-002)
  - No cookies set by the site itself (Vercel Analytics is cookieless)
- [ ] AC-007: **SEO sanity**:
  - `/rss.xml` validates against validator.w3.org/feed
  - `/sitemap.xml` validates
  - Each page has a unique title and description
  - OG image renders correctly when sharing to a social platform (use opengraph.xyz as a test tool)
- [ ] AC-008: **Operational**:
  - Anthropic billing alert set (Maria confirms)
  - Vercel error logging working — a test 500 produces a visible log entry (OPS-001)
  - Push-back Blob log accessible from Vercel dashboard
- [ ] AC-009: **Launch announcement ready** (optional): a Postcard #002 drafted *"the site is live, come argue with me"* — drafted in this story, shipped by Maria on launch day (or the day after) — optional per Maria's preference
- [ ] AC-010: **Known-issues list** documented in `docs/known-issues.md` — anything we consciously accepted as v1 debt (e.g. no RAG, no admin UI for push-back, no fancy OG image generation)
