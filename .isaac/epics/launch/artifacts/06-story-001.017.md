# Story 001.017: Domain cutover — bines.ai → Vercel

## Description

Point the `bines.ai` apex domain at the Vercel project by changing Name.com nameservers from the default (`ns1-4.name.com`) to Vercel's (`ns1.vercel-dns.com`, `ns2.vercel-dns.com`). This is a one-time, high-consequence Maria-executed action. Story documents the exact sequence, verifies SSL issuance, and sets up the www→apex redirect.

Critical: the order matters. The Vercel domain config must be ready before the nameservers change, otherwise the site is unreachable during the DNS propagation window (up to 48h).

## Acceptance Criteria

- [ ] AC-001: Vercel project has `bines.ai` configured as the primary domain (and `www.bines.ai` as a redirect source)
- [ ] AC-002: Vercel shows "valid configuration" for both `bines.ai` and `www.bines.ai` before nameservers are changed at Name.com
- [ ] AC-003: Name.com nameservers changed from `ns1-4.name.com` → `ns1.vercel-dns.com`, `ns2.vercel-dns.com` (4 removed, 2 added)
- [ ] AC-004: DNS propagation verified via `dig bines.ai NS` returning Vercel nameservers
- [ ] AC-005: `curl -I https://bines.ai` returns 200 with Vercel headers
- [ ] AC-006: `curl -I https://www.bines.ai` returns 308 redirecting to `https://bines.ai`
- [ ] AC-007: SSL cert issued by Vercel for both `bines.ai` and `www.bines.ai` (Let's Encrypt)
- [ ] AC-008: HTTPS-only enforced (HTTP redirects to HTTPS)
- [ ] AC-009: Production branch (`master`) deploys resolve at `bines.ai`
- [ ] AC-010: Cutover done during low-traffic window (UK evening); Maria available to rollback if needed

## Technical Tasks

1. Claude: update `docs/deploy-setup.md` with the domain cutover section — exact buttons to click at Vercel and Name.com
2. Maria: Vercel dashboard → project settings → Domains → add `bines.ai` and `www.bines.ai`
3. Vercel will instruct on DNS requirements — verify those are met before proceeding
4. Maria: Name.com → Manage Domain → Nameservers → Delete All → Add `ns1.vercel-dns.com`, `ns2.vercel-dns.com`
5. Both: wait for propagation (`dig +short bines.ai NS` until it shows `vercel-dns.com`; typically minutes, sometimes hours)
6. Vercel issues SSL cert automatically; wait for green
7. Verify with curl / browser
8. Claude: update `CLAUDE.md` — remove preview URL, add production URL

## Scope rules

- Do NOT add any custom DNS records at Name.com (no MX, no TXT for email) — v1 has no email on the domain
- Do NOT configure subdomains (no `blog.bines.ai`, no `chat.bines.ai`) — apex + www only
- Do NOT do this during peak hours — low-risk window only
- Keep a note of the original Name.com nameservers in case rollback is needed within the first 48h
