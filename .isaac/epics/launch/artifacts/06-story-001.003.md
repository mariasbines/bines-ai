# Story 001.003: Content pipeline — types, frontmatter schema, MDX loaders

## Description

Build the MDX content pipeline: Zod schemas for frontmatter validation, `gray-matter` frontmatter parsing, and typed loader functions that read MDX files from `content/` and return parsed + validated content objects. This is the data layer — everything downstream (Fieldwork rendering, Postcard rendering, archive grouping, /now, taste shelf, site stats) depends on it. Break this and everything breaks.

## Acceptance Criteria

- [ ] AC-001: Zod schemas defined for `FieldworkFrontmatter`, `PostcardFrontmatter`, `NowFrontmatter`, `TasteFrontmatter` matching the architecture doc data model exactly (field names, required/optional, enum values for status)
- [ ] AC-002: `getAllFieldwork()` reads all `.mdx` in `content/fieldwork/`, parses frontmatter, validates via Zod, returns typed array sorted by `published` descending
- [ ] AC-003: `getFieldworkBySlug(slug)` returns a single piece with compiled MDX content (using `@next/mdx`)
- [ ] AC-004: Equivalent loaders for Postcards, /now, taste — all typed, all Zod-validated
- [ ] AC-005: Invalid frontmatter throws a build-time error with file path + Zod error message (not a silent pass)
- [ ] AC-006: `getSiteStats()` returns `{ fieldworkCount, postcardCount, changedMyMindCount, updated }` derived from content loaders — feeds the `<CurrentlyStrip>` props in 001.002
- [ ] AC-007: `getCurrentlyLine()` reads `content/now.mdx` frontmatter and returns the `currently` field (used by `<CurrentlyStrip>`)
- [ ] AC-008: Fieldwork status enum: `'in-rotation' | 'retired-still-right' | 'retired-evolved' | 'changed-my-mind'` — validated strictly
- [ ] AC-009: Tags array validated as non-empty lowercase-kebab strings
- [ ] AC-010: All content loaders are server-side only (no client bundle pollution)

## Technical Tasks

2. Write `src/lib/content/fieldwork.ts` — loader functions: `getAllFieldwork`, `getFieldworkBySlug`, `getFieldworkByStatus`
3. Write `src/lib/content/postcards.ts` — `getAllPostcards`, `getPostcardByNumber`
4. Write `src/lib/content/now.ts` — `getNow()` returning parsed `/now`
5. Write `src/lib/content/taste.ts` — `getTaste()` returning the taste shelf
6. Write `src/lib/content/stats.ts` — `getSiteStats()` that composes other loaders
7. Write `src/lib/content/antipatterns.ts` — already created in 001.002; re-export from a content barrel if helpful
8. Create `content/` directory at repo root with `fieldwork/`, `postcards/` subdirs (empty in this story; seeded in 001.005 and 001.007)
9. Set up MDX serialization helper (use `@next/mdx` or `next-mdx-remote` as chosen in arch — default to `@next/mdx` since it's the Next-native path)
10. Add a content gitignore allowlist if needed (we want MDX files tracked)

## Scope rules

- No content authored in this story. Empty content dirs + `.gitkeep`.
- No UI components. Loaders only.
- MDX must be loaded from repo filesystem (`fs.readFile`), not fetched. This is SSG content.
- Do NOT expose any loader to the client — all are `import 'server-only'` or checked by convention.

## Risk mitigations

- **Frontmatter as data-boundary**: Zod validation here is the defence against malformed content breaking build. Non-zero-exit on invalid frontmatter is an AC (AC-005).
